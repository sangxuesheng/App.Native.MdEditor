# 🔍 滚动问题调试清单

## 请确认以下问题：

### 1. 是否有足够的内容？
- ❓ 编辑器中是否有超过屏幕高度的内容？
- ❓ 预览区域是否有超过屏幕高度的内容？
- 💡 如果内容不够多，不会显示滚动条

### 2. 是否刷新了页面？
- ❓ 修改后是否按 Ctrl+F5 强制刷新？
- ❓ 是否清除了浏览器缓存？
- 💡 浏览器可能缓存了旧的 JS/CSS 文件

### 3. 是否重新安装了 FPK？
- ❓ 是否使用 appcenter-cli 重新安装了 FPK？
- ❓ 是否重启了应用？
- 💡 可能还在运行旧版本的应用

### 4. 具体的"不能滑动"是指？
- ❓ 完全没有滚动条？
- ❓ 有滚动条但不能拖动？
- ❓ 鼠标滚轮不起作用？
- ❓ 触摸板滑动不起作用？
- ❓ 内容被截断看不到？

### 5. 哪个区域不能滑动？
- ❓ 编辑器区域（Monaco Editor）？
- ❓ 预览区域（Markdown 渲染）？
- ❓ 两个都不能滑动？

### 6. 布局模式
- ❓ 当前是什么布局？（垂直分屏/仅编辑器/仅预览）
- ❓ 是否显示了文件树？
- ❓ 是否显示了编辑器工具栏？

## 调试步骤

### 步骤 1: 测试预览区域滚动
1. 切换到"仅预览"模式
2. 在编辑器中输入大量内容（例如 50 行）
3. 查看预览区域是否可以滚动

### 步骤 2: 测试编辑器滚动
1. 切换到"仅编辑器"模式
2. 输入大量内容（例如 50 行）
3. 查看编辑器是否可以滚动

### 步骤 3: 测试垂直分屏滚动
1. 切换到"垂直分屏"模式
2. 输入大量内容
3. 分别测试左右两侧是否可以滚动

### 步骤 4: 检查浏览器控制台
1. 打开浏览器开发者工具（F12）
2. 查看 Console 是否有错误
3. 查看 Elements 面板，检查 DOM 结构

### 步骤 5: 检查 CSS
在浏览器开发者工具中检查以下元素的样式：
- `.app` - 应该有 `height: 100vh`
- `.main-content` - 应该有 `flex: 1`
- `.editor-preview-container` - 应该有 `flex: 1, min-height: 0`
- `.editor-pane` - 应该有 `flex: 1, min-height: 0`
- `.preview-pane` - 应该有 `flex: 1, min-height: 0, overflow: auto`

## 可能的原因

### 原因 1: 浏览器缓存
**解决方案**: Ctrl+Shift+Delete 清除缓存，或使用隐私模式

### 原因 2: 内容不够多
**解决方案**: 输入更多内容进行测试

### 原因 3: FPK 没有重新安装
**解决方案**: 
```bash
appcenter-cli uninstall App.Native.MdEditor2
appcenter-cli install App.Native.MdEditor2.fpk
```

### 原因 4: Monaco Editor 配置问题
**解决方案**: 已添加 scrollbar 配置

### 原因 5: CSS overflow 问题
**解决方案**: 已修改 .editor-pane 的 overflow 为 visible

### 原因 6: 高度计算问题
**解决方案**: 已添加 editor-preview-container 包装容器

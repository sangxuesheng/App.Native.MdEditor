## 🐛 Bug 修复

### 1. 文件清空问题分析与预防

**问题描述**：
在开发过程中，多次遇到文件被意外清空的情况，影响开发效率和数据安全。

**根本原因**：
- Cursor IDE 的文件操作工具（StrReplace/Write）在某些情况下存在 bug
- 当字符串匹配失败或操作异常时，可能导致文件被清空

**解决方案**：
- ✅ 使用 Git 版本控制进行文件保护
- ✅ 改用 shell 命令（sed、cat）进行文件操作
- ✅ 每次重要修改前先提交 Git
- ✅ 定期备份重要文件

**预防措施**：
```bash
# 修改前先提交
git add .
git commit -m "保存当前状态"

# 使用 sed 安全修改文件
sed -i 's/old/new/g' file.txt

# 如果文件被清空，立即恢复
git checkout file.txt
```

### 2. 移动端访问问题修复

**问题描述**：
在飞牛NAS手机客户端访问应用时，报错"The network connection was lost"。

**修复内容**：
- ✅ 将 `config.json` 中的 `protocol` 从 `"http"` 改为 `""` (自适应协议)
- ✅ 将 `type` 从 `"iframe"` 改为 `"url"` (新标签页打开)

**配置文件**：
```json
{
  "protocol": "",
  "type": "url",
  "port": 18080
}
```

**状态**：已修复，建议用户在移动端测试验证

### 3. 大文件加载优化

**当前实现**：
- 文件大小 > 1MB：提示用户确认
- 文件大小检查和提示机制

**优化建议**（待实现）：
- 分级提示：512KB、1MB、2MB 三个级别
- 大文件延迟加载：避免阻塞 UI
- 虚拟滚动：优化大文件渲染性能
- 禁用实时预览：减少性能开销

# 可拉缩面板功能 - 快速测试指南

## 测试前准备
1. 确保已备份原文件（已完成）
2. 重新构建前端应用
3. 启动应用服务器

## 测试步骤

### 测试 1: 文件树宽度调整
1. 打开应用，确保文件树可见
2. 将鼠标移动到文件树右侧边缘（应该看到一条细线）
3. 鼠标悬停时，分隔条应该高亮显示（蓝色/紫色）
4. 按住鼠标左键并向左/右拖动
5. 观察文件树宽度是否实时变化
6. 释放鼠标，宽度应该保持
7. 刷新页面，宽度应该保持不变（localStorage 持久化）

**预期结果**:
- 文件树宽度可在 200px - 600px 之间调整
- 拖拽时光标显示为 ↔
- 宽度变化流畅无卡顿

### 测试 2: 编辑器宽度调整（垂直布局）
1. 确保布局为"垂直"模式（编辑器和预览区左右排列）
2. 将鼠标移动到编辑器右侧边缘
3. 按住鼠标左键并拖动
4. 观察编辑器和预览区宽度是否同步变化
5. 刷新页面验证持久化

**预期结果**:
- 编辑器宽度可在 20% - 80% 之间调整
- 预览区自动占据剩余空间
- 拖拽流畅

### 测试 3: 编辑器高度调整（水平布局）
1. 切换到"水平"布局（编辑器和预览区上下排列）
2. 将鼠标移动到编辑器下方边缘
3. 按住鼠标左键并上下拖动
4. 观察编辑器和预览区高度是否同步变化

**预期结果**:
- 分隔条方向自动切换为水平
- 拖拽时光标显示为 ↕
- 高度调整流畅

### 测试 4: 主题切换
1. 在 Light 主题下测试分隔条颜色（应为 #0969da）
2. 切换到 Dark 主题，分隔条颜色应为 #58a6ff
3. 切换到 MD3 主题，分隔条颜色应为 #6750a4
4. 确保所有主题下拖拽功能正常

### 测试 5: 边界测试
1. 尝试将文件树拖到最小（200px）
2. 尝试将文件树拖到最大（600px）
3. 尝试将编辑器拖到最小（20%）
4. 尝试将编辑器拖到最大（80%）
5. 确认无法超出限制范围

### 测试 6: 布局切换
1. 在垂直布局下调整面板宽度
2. 切换到水平布局，观察是否正常
3. 切换到"仅编辑器"模式，只有文件树分隔条可用
4. 切换到"仅预览"模式，只有文件树分隔条可用

### 测试 7: 隐藏文件树
1. 点击隐藏文件树按钮
2. 确认文件树和其分隔条都消失
3. 编辑器和预览区应该占据全部空间
4. 重新显示文件树，宽度应该恢复

### 测试 8: 窗口大小变化
1. 调整浏览器窗口大小
2. 观察面板是否正确响应
3. 拖拽功能应该继续正常工作

## 常见问题排查

### 问题 1: 分隔条不显示
- 检查 Resizer.jsx 和 Resizer.css 是否正确创建
- 检查浏览器控制台是否有错误

### 问题 2: 拖拽无响应
- 检查 handleFileTreeResize 和 handleEditorResize 函数是否正确定义
- 检查 onResize 回调是否正确传递

### 问题 3: 宽度不持久化
- 检查浏览器是否支持 localStorage
- 打开浏览器开发者工具 -> Application -> Local Storage
- 查看是否有 md-editor-filetree-width 和 md-editor-editor-width 键

### 问题 4: 拖拽时页面卡顿
- 检查是否有大量的重渲染
- 考虑添加节流（throttle）优化

# 图片上传功能测试说明

## 📋 测试前准备

### 1. 构建前端（需要 Node.js 环境）
```bash
cd app/ui/frontend
npm install  # 如果是首次构建
npm run build
```

### 2. 启动服务器
```bash
cd /vol4/1000/开发文件夹/mac
./app/server/server.js
```

或者使用 FPK 包启动

---

## 🧪 测试步骤

### 测试 1: 打开图片管理器
1. 打开 Markdown 编辑器
2. 点击工具栏的"图片"按钮（或菜单：插入 → 图片）
3. ✅ 应该弹出"图片管理"对话框
4. ✅ 应该看到 4 个标签页：上传图片、图片链接、图片库、图床设置

### 测试 2: 上传图片（拖拽）
1. 切换到"上传图片"标签页
2. 准备一张测试图片（JPG/PNG/GIF/WebP，小于 10MB）
3. 拖拽图片到上传区域
4. ✅ 上传区域应该高亮显示
5. ✅ 松开鼠标后开始上传
6. ✅ 上传成功后显示在"最近上传"区域
7. ✅ 图片应该有"插入"按钮

### 测试 3: 上传图片（点击选择）
1. 点击"选择图片"按钮
2. 在文件选择器中选择一张或多张图片
3. ✅ 上传成功后显示在"最近上传"区域

### 测试 4: 插入上传的图片
1. 在"最近上传"区域找到一张图片
2. 点击图片上的"插入"按钮
3. ✅ 对话框应该关闭
4. ✅ 编辑器中应该插入 Markdown 语法：`![图片描述](图片URL)`
5. ✅ 预览区域应该显示图片

### 测试 5: 使用图片链接
1. 切换到"图片链接"标签页
2. 输入图片 URL：`https://picsum.photos/200/300`
3. 输入图片描述：`测试图片`
4. 点击"插入图片"按钮
5. ✅ 对话框应该关闭
6. ✅ 编辑器中应该插入：`![测试图片](https://picsum.photos/200/300)`

### 测试 6: 图片库（占位）
1. 切换到"图片库"标签页
2. ✅ 应该显示"图片库为空"的提示

### 测试 7: 图床设置（占位）
1. 切换到"图床设置"标签页
2. ✅ 应该显示"此功能正在开发中..."的提示
3. ✅ 应该列出未来支持的图床类型

### 测试 8: 主题切换
1. 切换编辑器主题（浅色/深色/MD3）
2. 打开图片管理器
3. ✅ 对话框样式应该跟随主题变化

### 测试 9: 文件类型验证
1. 尝试上传非图片文件（如 .txt, .pdf）
2. ✅ 应该显示错误提示

### 测试 10: 文件大小验证
1. 尝试上传超过 10MB 的图片
2. ✅ 应该显示错误提示

## 🐛 常见问题

### 问题 1: 对话框不显示
**原因**: 前端未重新构建  
**解决**: 运行 `npm run build`

### 问题 2: 上传失败
**原因**: 后端 API 未启动或路径权限问题  
**解决**: 检查服务器日志，确认授权目录配置

### 问题 3: 图片不显示
**原因**: 图片静态服务未正确配置  
**解决**: 检查 `/images/*` 路由是否正常工作

### 问题 4: 样式错乱
**原因**: CSS 文件未正确加载  
**解决**: 检查 `ImageManagerDialog.css` 是否存在

# 🎯 代码行号显示指南

## ✅ 问题解决进度

- ✅ **代码块靠右问题** - 已修复
- ⏳ **行号不显示问题** - 需要手动启用

## 📝 启用代码行号的步骤

### 方法 1: 通过导出配置面板

1. **打开导出配置面板**
   - 在右侧工具栏找到"导出配置"按钮
   - 点击打开配置面板

2. **启用代码块行号**
   - 在配置面板中找到"代码块行号"选项
   - 勾选该选项（checkbox）

3. **查看预览**
   - 预览区域的代码块应该立即显示行号
   - 如果没有显示，刷新页面

4. **导出 HTML**
   - 点击导出按钮
   - 选择 HTML 格式
   - 导出的 HTML 将包含行号

### 方法 2: 通过浏览器控制台（临时测试）

如果你想快速测试，可以在浏览器控制台中运行：

```javascript
// 查看当前配置
console.log('当前行号配置:', exportConfig.codeLineNumbers);

// 临时启用行号（仅用于测试，刷新后失效）
// 注意：这个方法可能不起作用，因为 exportConfig 可能不在全局作用域
```

## 🔍 验证行号是否启用

### 检查控制台日志

启用行号后，控制台应该显示：

```
[代码行号] exportConfig.codeLineNumbers: true
[代码行号] 开始处理代码块
[代码行号] 找到代码块数量: 3
[代码行号] 代码块 1 有 5 行
[代码行号] 代码块 1 处理完成
```

如果看到：
```
[代码行号] exportConfig.codeLineNumbers: false
```

说明行号功能没有启用。

### 检查预览效果

启用行号后，代码块应该显示为：

```
  1  function greet(name) {
  2    console.log(`Hello, ${name}!`);
  3    return `Welcome to Markdown Editor`;
  4  }
  5  
  6  const message = greet('用户');
```

## 🎨 行号样式说明

### 双列布局

启用行号后，代码块使用双列布局：

```
┌─────────────────────────────────┐
│ 行号列 │ 代码列                 │
├────────┼────────────────────────┤
│   1    │ function hello() {     │
│   2    │   console.log('Hi');   │
│   3    │ }                      │
└────────┴────────────────────────┘
```

### 行号特性

- ✅ 右对齐
- ✅ 灰色显示（#666）
- ✅ 不可选择（复制代码时不包含行号）
- ✅ 右侧有分隔线
- ✅ 自动编号（从 1 开始）

## 🐛 常见问题

### Q1: 勾选了"代码块行号"但预览中没有显示？

**可能原因：**
1. 代码块只有 1 行（自动跳过）
2. 是 Mermaid 图表（自动跳过）
3. 页面需要刷新

**解决方案：**
1. 确保代码块有多行
2. 刷新页面（F5）
3. 查看控制台是否有错误

### Q2: 预览中有行号，但导出的 HTML 没有？

**可能原因：**
1. 导出时没有启用行号
2. 浏览器缓存了旧的 HTML

**解决方案：**
1. 确认导出配置中"代码块行号"已勾选
2. 清除浏览器缓存（Ctrl + Shift + Delete）
3. 重新导出

### Q3: 行号显示但代码还是靠右？

**可能原因：**
1. 浏览器缓存了旧的 CSS
2. 需要强制刷新

**解决方案：**
1. 清除浏览器缓存
2. 强制刷新（Ctrl + F5）
3. 重新导出 HTML

### Q4: 复制代码时包含了行号？

**可能原因：**
1. CSS 的 `user-select: none` 没有生效

**解决方案：**
1. 检查浏览器是否支持 `user-select`
2. 尝试在不同浏览器中测试

## 📊 配置文件位置

在 `App.jsx` 第 146 行：
```javascript
codeLineNumbers: false,  // 默认不启用
```

### 配置面板

在 `ExportConfigPanel.jsx` 第 979-981 行：
```javascript
label="代码块行号"
checked={config.codeLineNumbers || false}
onChange={(value) => updateConfig('codeLineNumbers', value)}
```

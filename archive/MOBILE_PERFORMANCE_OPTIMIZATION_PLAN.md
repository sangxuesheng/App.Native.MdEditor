# 移动端流畅性优化方案 - 系统性整理

## 🎯 优化目标

**接近原生系统的流畅度**
- 目标帧率：60 FPS（16.67ms/帧）
- 触摸响应：<100ms
- 滚动流畅：无卡顿、无掉帧
- 动画流畅：硬件加速、GPU 渲染

---

## 📊 优化分类体系

### 一、渲染性能优化（核心）

#### 1.1 虚拟滚动（Virtual Scrolling）
**优化点**：文件树、历史记录等长列表
**当前问题**：
- 文件树渲染所有节点，节点多时卡顿
- 历史记录一次性渲染所有项
- DOM 节点过多导致重排重绘

**优化方案**：
- 只渲染可视区域的节点
- 动态加载/卸载不可见节点
- 使用 `react-window` 或 `react-virtualized`

**预期提升**：
- 渲染时间：减少 70-90%
- 内存占用：减少 60-80%
- 滚动帧率：从 30-40 FPS → 60 FPS
- 首次渲染：从 500ms+ → 100ms

---

#### 1.2 懒加载与代码分割
**优化点**：组件、图片、Monaco Editor
**当前问题**：
- Monaco Editor 体积大（~500KB），首次加载慢
- 所有组件一次性加载
- 图片未懒加载

**优化方案**：
- Monaco Editor 懒加载（已部分实现）
- 对话框组件按需加载
- 图片懒加载 + IntersectionObserver
- 路由级代码分割

**预期提升**：
- 首屏加载：减少 40-50%
- 初始 JS 体积：减少 30-40%
- Time to Interactive：从 3s → 1.5s
- 内存占用：减少 30%

---

#### 1.3 防抖与节流
**优化点**：输入、滚动、窗口调整
**当前问题**：
- 编辑器输入触发频繁的 Markdown 处理
- 滚动事件未节流
- 窗口 resize 频繁触发重新计算

**优化方案**：
- 编辑器输入防抖（300ms）
- 滚动事件节流（16ms）
- resize 事件节流（100ms）
- 使用 `requestAnimationFrame`

**预期提升**：
- CPU 占用：减少 50-70%
- 输入响应：更流畅，无卡顿
- 滚动性能：从 40 FPS → 60 FPS
- 电池续航：提升 20-30%

---

#### 1.4 Markdown 渲染优化
**优化点**：实时预览处理
**当前问题**：
- 每次输入都重新处理整个文档
- 大文档处理耗时长（>500ms）
- 阻塞主线程

**优化方案**：
- 增量渲染：只处理变化的部分
- Web Worker 处理 Markdown
- 虚拟 DOM diff 优化
- 缓存处理结果

**预期提升**：
- 处理时间：减少 60-80%
- 主线程阻塞：从 500ms → 50ms
- 输入延迟：从 200ms → 50ms
- 大文档性能：提升 5-10 倍

---

### 二、CSS 与动画优化

#### 2.1 硬件加速
**优化点**：所有动画和过渡
**当前问题**：
- 部分动画使用 CPU 渲染
- 未使用 `transform` 和 `opacity`
- 触发重排重绘

**优化方案**：
- 使用 `transform` 替代 `left/top`
- 使用 `opacity` 替代 `visibility`
- 添加 `will-change` 提示
- 使用 `translate3d` 强制 GPU 加速

**预期提升**：
- 动画帧率：从 30-40 FPS → 60 FPS
- CPU 占用：减少 40-60%
- 动画流畅度：接近原生
- 电池续航：提升 15-25%

---

#### 2.2 减少重排重绘
**优化点**：DOM 操作、样式变更
**当前问题**：
- 频繁的 DOM 读写混合
- 样式批量修改触发多次重排
- 布局抖动（Layout Thrashing）

**优化方案**：
- 批量 DOM 操作
- 使用 `DocumentFragment`
- 读写分离（先读后写）
- 使用 CSS `contain` 属性

**预期提升**：
- 重排次数：减少 70-90%
- 渲染时间：减少 50-70%
- 滚动性能：提升 2-3 倍
- 交互响应：从 150ms → 50ms

---

#### 2.3 CSS 优化
**优化点**：选择器、样式计算
**当前问题**：
- 复杂的 CSS 选择器
- 未使用 CSS containment
- 样式计算耗时

**优化方案**：
- 简化选择器（避免深层嵌套）
- 使用 `contain: layout style paint`
- 使用 CSS 变量减少重复计算
- 移除未使用的 CSS

**预期提升**：
- 样式计算：减少 30-50%
- 首次渲染：减少 20-30%
- 重绘时间：减少 40-60%

---

### 三、触摸与交互优化

#### 3.1 触摸响应优化
**优化点**：按钮、菜单、滚动
**当前问题**：
- 300ms 点击延迟（部分场景）
- 触摸反馈不明显
- 滚动惯性不自然

**优化方案**：
- 使用 `touch-action: manipulation`
- 添加触摸反馈动画（<100ms）
- 优化滚动容器（`-webkit-overflow-scrolling: touch`）
- 使用 Passive Event Listeners

**预期提升**：
- 点击响应：从 300ms → 50ms
- 触摸反馈：即时可见
- 滚动流畅度：接近原生
- 用户体验：显著提升

- 添加手势反馈

**预期提升**：

- 用户体验：更自然流畅

## 🔧 技术实施细节

### 1. 防抖与节流实现

```javascript
// 防抖：编辑器输入
const debouncedUpdate = useMemo(
  () => debounce((value) => {
    // 处理 Markdown
    processMarkdown(value)
  }, 300),
  []
)

// 节流：滚动事件
const throttledScroll = useMemo(
  () => throttle((e) => {
    // 处理滚动
    handleScroll(e)
  }, 16), // 60 FPS
  []
)
```

### 2. 硬件加速

```css
/* 使用 transform 替代 left/top */
.element {
  transform: translate3d(0, 0, 0);
  will-change: transform;
}

/* 动画优化 */
@keyframes slide {
  from { transform: translateX(-100%); }
  to { transform: translateX(0); }
}
```

### 3. 虚拟滚动

```javascript
import { FixedSizeList } from 'react-window'

<FixedSizeList
  height={600}
  itemCount={items.length}
  itemSize={35}
  width="100%"
>
  {Row}
</FixedSizeList>
```

### 4. Web Worker

```javascript
// markdown.worker.js
self.onmessage = (e) => {
  const html = processMarkdown(e.data)
  self.postMessage(html)
}

// 主线程
const worker = new Worker('markdown.worker.js')
worker.postMessage(content)
worker.onmessage = (e) => setHtml(e.data)
```

# 移动端回归检查清单

适用范围：
- 开发态：`http://NAS_IP:3000`
- 安装态：`http://NAS_IP:18080`

建议测试环境：
- iPhone Safari
- 飞牛移动端容器
- 至少覆盖一次竖屏和横屏

## 一、基础访问

- [ ] `3000` 开发态可以从手机局域网直接访问
- [ ] `18080` 安装态可以从手机局域网直接访问
- [ ] 首屏不白屏，不出现资源 404
- [ ] 刷新页面后仍能正常进入应用

## 二、布局与安全区

- [ ] 顶部工具栏没有被刘海或状态栏遮挡
- [ ] 底部状态栏没有被手势条遮挡
- [ ] 横屏时左右两侧没有内容贴边或被裁切
- [ ] 打开文件树、导出配置、设置弹层时，四周留白正常

## 三、文件树与操作菜单

- [ ] 点击文件可以正常打开
- [ ] 点击目录箭头可以正常展开/折叠
- [ ] 长按节点只出现应用内菜单，不出现系统“复制”气泡
- [ ] 点击“更多”按钮可以稳定打开菜单
- [ ] 文件树侧栏可以正常关闭

## 四、编辑与预览

- [ ] 手机竖屏下可以在“编辑/预览”之间切换
- [ ] 平板或较宽视口下仍保持合理的双栏/抽屉式布局
- [ ] 软键盘弹出后，编辑区仍可见，不会只剩半屏
- [ ] 保存、切换文件、打开历史记录等高频操作可正常完成

## 五、内容阅读

- [ ] 代码块可以横向滚动，不会把页面挤坏
- [ ] 表格可以横向浏览，不会超出容器后消失
- [ ] Mermaid 图表可以完整查看
- [ ] 图片不会超出屏幕宽度
- [ ] 带内联宽度的图片在手机上仍能被完整查看

## 六、弹层与反馈

- [ ] 设置、历史记录、确认框在手机上显示完整
- [ ] 点击遮罩可按预期关闭弹层
- [ ] `Esc` 可以关闭确认框（桌面调试时验证）
- [ ] 失败和成功提示优先使用应用内 toast，而不是系统弹窗

## 七、开发态与安装态一致性

- [ ] 在 `3000` 下验证通过的文件树交互，在 `18080` 下表现一致
- [ ] 在 `3000` 下验证通过的弹层布局，在 `18080` 下表现一致
- [ ] 在 `3000` 下验证通过的代码块/表格/图片阅读，在 `18080` 下表现一致
- [ ] 前端重新构建后，安装态资源路径和样式正常

## 推荐执行顺序

1. 先在 `3000` 开发态完成真机快速回归。
2. 通过后执行前端构建，并切到 `18080` 安装态复测。
3. 若修改了入口、端口或打包相关配置，再补做完整安装链路验证。

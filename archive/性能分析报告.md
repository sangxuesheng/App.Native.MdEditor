## 🚨 主要性能问题

### 1. App.jsx 过于庞大 ⚠️⚠️⚠️

**问题**：
- 3,680 行代码集中在单个文件
- 40+ 个 state 变量
- 30+ 个 useEffect
- 所有业务逻辑混在一起

**影响**：
- 组件重渲染频繁
- 代码难以维护
- 性能优化困难
- 开发体验差

### 2. 缺少性能优化 ⚠️⚠️

**问题**：
- useCallback/useMemo 使用不足
- 没有 React.memo 包裹子组件
- 大量不必要的重渲染
- 没有虚拟滚动

**影响**：
- 编辑时卡顿
- 预览更新慢
- 内存占用高

### 3. 构建产物过大 ⚠️

**问题**：
- markdown-vendor 764KB
- index.js 444KB（包含 Monaco）
- 总计 2.7MB

**影响**：
- 首次加载慢
- 网络传输时间长
- 移动端体验差

### 4. Monaco Editor 未优化 ⚠️

**问题**：
- Monaco Editor 完整打包
- 包含所有语言支持
- 未按需加载

**影响**：
- 增加 ~2MB 体积
- 加载时间长

## 💡 优化建议

### 优先级 1：拆分 App.jsx（重要！）

#### 方案：按功能模块拆分

```
src/
├── App.jsx (主入口，200 行以内)
├── hooks/
│   ├── useEditor.js          # 编辑器逻辑
│   ├── useFileOperations.js  # 文件操作
│   ├── useMarkdown.js        # Markdown 处理
│   ├── useImageManager.js    # 图片管理
│   └── useExport.js          # 导出功能
├── contexts/
│   ├── EditorContext.jsx     # 编辑器状态
│   └── ThemeContext.jsx      # 主题状态
└── components/
    ├── Editor/
    │   ├── EditorPanel.jsx
    │   └── EditorToolbar.jsx
    ├── Preview/
    │   └── PreviewPanel.jsx
    └── ...
```

**预期效果**：
- App.jsx 从 3,680 行 → 200 行
- 代码可维护性提升 80%
- 重渲染减少 50%

#### 实施步骤：

1. **第一步：提取自定义 Hooks**
   ```javascript
   // hooks/useEditor.js
   export function useEditor(initialContent) {
     const [content, setContent] = useState(initialContent)
     const [currentPath, setCurrentPath] = useState('')
     // ... 编辑器相关逻辑
     return { content, setContent, currentPath, ... }
   }
   ```

2. **第二步：创建 Context**
   ```javascript
   // contexts/EditorContext.jsx
   export const EditorContext = createContext()
   export function EditorProvider({ children }) {
     const editor = useEditor()
     return <EditorContext.Provider value={editor}>{children}</EditorContext.Provider>
   }
   ```

3. **第三步：拆分组件**
   ```javascript
   // components/Editor/EditorPanel.jsx
   export function EditorPanel() {
     const { content, setContent } = useContext(EditorContext)
     return <Editor value={content} onChange={setContent} />
   }
   ```

### 优先级 2：性能优化

#### 2.1 使用 React.memo

```javascript
// 包裹不需要频繁更新的组件
export const FileTree = React.memo(FileTreeComponent)
export const EditorToolbar = React.memo(EditorToolbarComponent)
export const PreviewPanel = React.memo(PreviewPanelComponent)
```

#### 2.2 使用 useCallback

```javascript
// 缓存回调函数
const handleSave = useCallback(async () => {
  await saveFile(currentPath, content)
}, [currentPath, content])

const handleExport = useCallback(async (format) => {
  // 导出逻辑
}, [content])
```

#### 2.3 使用 useMemo

```javascript
// 缓存计算结果
const markdownHtml = useMemo(() => {
  return processor.processSync(content).toString()
}, [content])

const fileTreeData = useMemo(() => {
  return buildFileTree(files)
}, [files])
```

#### 2.4 防抖优化

```javascript
// 已有 useDebounce，确保正确使用
const debouncedContent = useDebounce(content, 500) // 500ms 防抖

useEffect(() => {
  // 使用防抖后的内容更新预览
  updatePreview(debouncedContent)
}, [debouncedContent])
```

### 优先级 3：减小构建体积

#### 3.1 Monaco Editor 按需加载

```javascript
// vite.config.js
import monacoEditorPlugin from 'vite-plugin-monaco-editor'

export default defineConfig({
  plugins: [
    react(),
    monacoEditorPlugin({
      // 只加载需要的语言
      languages: ['markdown', 'javascript', 'typescript', 'json'],
      // 只加载需要的功能
      features: ['coreCommands', 'find']
    })
  ]
})
```

**预期效果**：减少 1-1.5MB

#### 3.2 动态导入重量级库

```javascript
// 懒加载 Mermaid
const MermaidPreview = lazy(() => import('./components/MermaidPreview'))

// 懒加载导出功能
const ExportDialog = lazy(() => import('./components/ExportDialog'))

// 使用 Suspense
<Suspense fallback={<Loading />}>
  <MermaidPreview />
</Suspense>
```

#### 3.3 优化 Markdown 处理

```javascript
// 考虑使用更轻量的库
// 或者按需加载 rehype/remark 插件
const processor = useMemo(() => {
  return unified()
    .use(remarkParse)
    .use(remarkGfm)
    // 只在需要时加载数学公式
    .use(content.includes('$') ? remarkMath : null)
    .use(remarkRehype)
    // ...
}, [content])
```

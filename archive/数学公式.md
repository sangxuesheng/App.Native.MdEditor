# 数学公式在微信公众号格式中的处理

## 📍 概述

在导出微信公众号格式时，数学公式通过 **MathJax** 渲染为 **SVG 格式**，然后随 HTML 一起复制到剪贴板，确保在微信编辑器中正确显示。

## 🎯 核心技术

### 1. 数学公式渲染引擎

项目使用 **MathJax 3** 作为数学公式渲染引擎：

```html
<!-- index.html -->
<script>
  window.MathJax = {
    tex: { tags: 'ams' },
    svg: { fontCache: 'none' },
  }
</script>
<script
  id="MathJax-script"
  src="https://cdn-doocs.oss-cn-shenzhen.aliyuncs.com/npm/mathjax@3/es5/tex-svg.js"
></script>
```

**配置说明**：
- `tex: { tags: 'ams' }` - 使用 AMS 数学标签
- `svg: { fontCache: 'none' }` - 禁用字体缓存，确保每次渲染独立

## 🏗️ 实现架构

```
Markdown 数学公式
    ↓
MDKatex 扩展（marked.js）
    ↓
识别公式语法
    ├─ $...$ (行内公式)
    ├─ $$...$$ (块级公式)
    ├─ \(...\) (LaTeX 行内)
    └─ \[...\] (LaTeX 块级)
    ↓
MathJax 渲染
    ├─ tex2svg() 转换
    └─ 生成 SVG
    ↓
包装为 HTML
    ├─ 行内: <span class="katex-inline">
    └─ 块级: <section class="katex-block">
    ↓
复制到微信
    ↓
SVG 在微信中显示 ✅
```

## 📝 核心实现

### 1. MDKatex 扩展

**位置**: `packages/core/src/extensions/katex.ts`

```typescript
export function MDKatex(
  options: MarkedKatexOptions | undefined, 
  withStyle: boolean = true
): MarkedExtension {
  return {
    extensions: [
      inlineKatex(options, createRenderer(false, withStyle)),      // $...$
      blockKatex(options, createRenderer(true, withStyle)),        // $$...$$
      inlineLatexKatex(options, createRenderer(false, withStyle)), // \(...\)
      blockLatexKatex(options, createRenderer(true, withStyle)),   // \[...\]
    ],
  }
}
```

### 2. 公式语法识别

#### 行内公式（$...$）

```typescript
// 标准模式：前后需要空格或标点
const inlineRule = /^(\${1,2})(?!\$)((?:\\.|[^\\\n])*?(?:\\.|[^\\\n$]))\1(?=[\s?!.,:？！。，：]|$)/

// 非标准模式：不需要空格
const inlineRuleNonStandard = /^(\${1,2})(?!\$)((?:\\.|[^\\\n])*?(?:\\.|[^\\\n$]))\1/
```

**示例**：
```markdown
这是行内公式 $E=mc^2$ 的例子。
```

#### 块级公式（$$...$$）

```typescript
const blockRule = /^\s{0,3}(\${1,2})[ \t]*\n([\s\S]+?)\n\s{0,3}\1[ \t]*(?:\n|$)/
```

**示例**：
```markdown
$$
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
$$
```

#### LaTeX 风格

```typescript
// 行内: \(...\)
const inlineLatexRule = /^\\\(([^\\]*(?:\\.[^\\]*)*?)\\\)/

// 块级: \[...\]
const blockLatexRule = /^\\\[([^\\]*(?:\\.[^\\]*)*?)\\\]/
```

**示例**：
```markdown
行内公式 \(E=mc^2\) 和块级公式：

\[
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
\]
```

### 3. 渲染器实现

```typescript
function createRenderer(defaultDisplay: boolean, withStyle: boolean = true) {
  return (token: any) => {
    const display = token.displayMode ?? defaultDisplay

    // 使用 MathJax 渲染
    window.MathJax.texReset()
    const mjxContainer = window.MathJax.tex2svg(token.text, { display })
    const svg = mjxContainer.firstChild
    
    // 获取宽度
    const width = svg.style['min-width'] || svg.getAttribute('width')
    svg.removeAttribute('width')

    // 设置样式（用于微信复制）
    if (withStyle) {
      svg.style.display = 'initial'
      svg.style.setProperty('max-width', '300vw', 'important')
      svg.style.flexShrink = '0'
      svg.style.width = width
    }

    // 返回 HTML
    if (!display) {
      // 行内公式
      return `<span class="katex-inline">${svg.outerHTML}</span>`
    }

    // 块级公式
    return `<section class="katex-block">${svg.outerHTML}</section>`
  }
}
```

### 4. 生成的 HTML 结构

#### 行内公式

```html
<span class="katex-inline">
  <svg xmlns="http://www.w3.org/2000/svg" 
       style="display: initial; max-width: 300vw !important; flex-shrink: 0; width: 5.5em;">
    <!-- SVG 内容 -->
    <g>
      <text>E = mc²</text>
    </g>
  </svg>
</span>
```

#### 块级公式

```html
<section class="katex-block">
  <svg xmlns="http://www.w3.org/2000/svg" 
       style="display: initial; max-width: 300vw !important; flex-shrink: 0; width: 15em;">
    <!-- SVG 内容 -->
    <g>
      <text>∫₋∞^∞ e⁻ˣ² dx = √π</text>
    </g>
  </svg>
</section>
```

## 🔄 完整流程

### 编辑阶段

```
1. 用户输入 Markdown
   $E=mc^2$
   ↓
2. marked.js 解析
   - MDKatex 扩展识别公式
   ↓
3. MathJax 渲染
   - tex2svg() 转换为 SVG
   ↓
4. 包装为 HTML
   <span class="katex-inline">
     <svg>...</svg>
   </span>
   ↓
5. 显示在预览区 ✅
```

### 复制到微信阶段

```
1. 用户点击"复制"
   ↓
2. processClipboardContent()
   - 获取 #output 的 innerHTML
   - 包含所有渲染后的 SVG
   ↓
3. 样式内联（juice）
   - SVG 的 style 属性保持不变
   ↓
4. 写入剪贴板
   - ClipboardItem API
   - text/html 格式
   ↓
5. 粘贴到微信编辑器
   - SVG 直接显示
   ↓
6. 公式正确显示 ✅
```

## 💡 关键技术点

### 1. SVG 格式的优势

**为什么使用 SVG 而不是图片？**

✅ **优点**：
- 矢量图形，无损缩放
- 文件体积小
- 可以直接嵌入 HTML
- 微信编辑器原生支持
- 不需要额外的图片服务器

❌ **如果使用图片**：
- 需要上传到图床
- 依赖外部服务
- 可能失效
- 体积更大

### 2. 样式处理

```typescript
// 行内公式对齐问题
// 直接设置 style 属性，确保在微信中正确显示
svg.style.display = 'initial'
svg.style.setProperty('max-width', '300vw', 'important')
svg.style.flexShrink = '0'
svg.style.width = width
```

**为什么这样设置？**
- `display: initial` - 确保行内显示
- `max-width: 300vw` - 允许公式超出容器宽度
- `flex-shrink: 0` - 防止公式被压缩
- `width: width` - 保持原始宽度

### 3. 字体缓存禁用

```javascript
window.MathJax = {
  svg: { fontCache: 'none' },
}
```

**为什么禁用字体缓存？**
- 确保每个公式独立渲染
- 避免缓存导致的样式问题
- 微信编辑器不支持共享字体缓存

### 4. 公式重置

```typescript
window.MathJax.texReset()
```

**作用**：
- 重置 TeX 状态
- 避免公式之间相互影响
- 确保每个公式独立

- 浅色主题：黑色文字

## 🐛 常见问题

### 1. 公式不显示

**问题**: 复制到微信后公式不显示

**原因**: 
- MathJax 未加载完成
- SVG 被过滤

**解决**:
```typescript
// 确保 MathJax 已加载
if (!window.MathJax) {
  console.error('MathJax 未加载')
  return
}
```

### 2. 公式对齐问题

**问题**: 行内公式与文字不对齐

**原因**: 
- SVG 的 display 属性不正确
- vertical-align 未设置

**解决**:
```css
.katex-inline svg {
  display: initial;
  vertical-align: middle;
}
```

### 3. 公式被截断

**问题**: 长公式在微信中被截断

**原因**: 
- 容器宽度限制
- max-width 设置不当

**解决**:
```typescript
svg.style.setProperty('max-width', '300vw', 'important')
```

### 4. 公式模糊

**问题**: 公式在微信中显示模糊

**原因**: 
- SVG 的 viewBox 问题
- 缩放比例不对

**解决**:
- MathJax 自动处理 viewBox
- 保持原始宽度不变

# 热更新快速参考

## 🚀 一键热更新

```bash
./hot-update.sh
```

选择更新方式：
1. 前端热更新 ⚡ - 最快（10-20秒）
2. 后端热更新 🔄 - 需重启（10秒）
3. 完整热更新 📦 - 全部更新（30秒）
4. 监听模式 👀 - 自动更新

## 📋 各脚本说明

### 前端热更新（推荐）
```bash
./hot-update-frontend.sh
```
- ✅ 无需重启应用
- ✅ 10-20秒完成
- ✅ 自动备份
- 🔄 需刷新浏览器（Ctrl+Shift+R）

### 后端热更新
```bash
./hot-update-backend.sh
```
- ⚠️ 需要重启应用
- ✅ 10秒完成
- ✅ 自动备份
- ✅ 自动验证服务

### 完整热更新
```bash
./hot-update-full.sh
```
- ⚠️ 需要重启应用
- ✅ 30秒完成
- ✅ 自动备份
- ✅ 失败自动回滚

### 监听模式（开发神器）
```bash
./hot-update-watch.sh
```
- ✅ 自动监听文件变化
- ✅ 自动构建和更新
- ✅ 实时开发体验
- 🛑 Ctrl+C 停止

```bash

```

## ⚡ 开发工作流

### 前端开发（推荐）
```bash
# 终端1：启动监听模式
./hot-update-watch.sh

# 终端2：编辑代码
vim app/ui/frontend/src/App.jsx

# 保存后自动更新，刷新浏览器即可
```

### 后端开发
```bash
# 编辑代码
vim app/server/server.js

# 执行热更新
./hot-update-backend.sh
```

### 全栈开发
```bash
# 编辑前端和后端
vim app/ui/frontend/src/App.jsx
vim app/server/server.js

# 执行完整热更新
./hot-update-full.sh
```

## ⚠️ 注意事项

1. **仅用于开发** - 不要在生产环境使用
2. **刷新浏览器** - 前端更新后需强制刷新
3. **定期备份** - 热更新会自动备份
4. **提交代码** - 测试通过后记得提交到 Git

## 🎯 效率对比

| 操作 | 传统方式 | 热更新 | 节省 |
|------|---------|--------|------|
| 前端 | 2-3分钟 | 10-20秒 | 90% |
| 后端 | 1-2分钟 | 10秒 | 85% |
| 完整 | 3-5分钟 | 30秒 | 90% |

## 🆘 故障排除

### 更新失败
```bash
# 查看备份
ls -lh backups/

# 手动回滚
cp -r backups/hot-update-xxx/dist /var/apps/App.Native.MdEditor2/target/ui/frontend/
```

### 服务异常
```bash
# 查看日志
tail -f /var/log/apps/App.Native.MdEditor2.log

# 重启应用
appcenter-cli stop App.Native.MdEditor2
appcenter-cli start App.Native.MdEditor2
```

### 浏览器缓存
```bash
# 强制刷新
Ctrl+Shift+R

# 或清除缓存
浏览器设置 → 清除缓存
```

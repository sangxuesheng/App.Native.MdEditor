# 自动保存功能实现文档

## 📋 功能概述

已完成自动保存机制的开发，包括：

### ✅ 核心功能

1. **定时自动保存**
   - 默认每 30 秒自动保存草稿到 localStorage
   - 可通过工具栏按钮开启/关闭自动保存
   - 使用防抖优化，避免频繁保存

2. **草稿管理**
   - 草稿存储在浏览器 localStorage
   - 支持多文件草稿管理
   - 自动清理旧草稿（保留最近 10 个）
   - localStorage 满时自动清理（保留最近 5 个）

3. **草稿恢复**
   - 打开文件时自动检测草稿
   - 弹出对话框提示恢复或丢弃
   - 显示草稿保存时间和预览
   - 恢复后自动清理草稿

4. **保存状态指示**
   - 工具栏显示未保存指示器（橙色圆点）
   - 状态栏显示"未保存"提示
   - 自动保存按钮状态切换
   - 保存成功后显示提示信息

5. **快捷键支持**
   - `Ctrl/Cmd + S`: 手动保存到服务器
   - 自动保存为草稿，手动保存到服务器并清除草稿

## 🔄 工作流程

### 1. 编辑文件
```
用户编辑 → 内容变化 → 30秒后自动保存草稿 → localStorage
```

### 2. 手动保存
```
Ctrl/Cmd + S → 保存到服务器 → 清除草稿 → 显示"文件已保存"
```

### 3. 打开文件
```
加载文件 → 检测草稿 → 有草稿？
                        ├─ 是 → 显示恢复对话框
                        └─ 否 → 直接加载文件
```

### 4. 恢复草稿
```
用户选择"恢复" → 加载草稿内容 → 重置保存状态 → 显示"已恢复草稿"
```

### 5. 丢弃草稿
```
用户选择"丢弃" → 清除草稿 → 显示"已丢弃草稿"
```

## 📊 技术细节

### localStorage 存储结构

```javascript
// 草稿数据
{
  "md-editor-draft:/path/to/file.md": {
    "filePath": "/path/to/file.md",
    "content": "# 文件内容...",
    "timestamp": 1771850000000,
    "size": 1234
  }
}

// 草稿索引
{
  "md-editor-drafts-index": [
    "/path/to/file1.md",
    "/path/to/file2.md"
  ]
}
```

### 自动保存配置

```javascript
{
  interval: 30000,        // 30 秒
  enabled: true,          // 默认开启
  onAutoSaveSuccess,      // 成功回调
  onAutoSaveError         // 错误回调
}
```

### 草稿清理策略

1. **正常清理**: 保留最近 10 个草稿
2. **空间不足**: 保留最近 5 个草稿
3. **手动保存**: 清除当前文件草稿

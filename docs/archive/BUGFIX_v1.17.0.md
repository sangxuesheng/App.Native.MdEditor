# v1.17.0 问题修复说明

## 修复的问题

### 1. ✅ 粘贴图片后文本无法粘贴
**问题描述**: 添加粘贴图片功能后，普通文本无法通过 Ctrl+V 粘贴到编辑器

**修复方案**: 
- 修改粘贴事件处理逻辑
- 只在剪贴板包含图片时才拦截事件
- 如果只有文本，让事件正常传递给编辑器

**修改文件**: `app/ui/frontend/src/App.jsx`

---

### 2. ✅ 图片压缩需要用户可配置
**问题描述**: 图片压缩功能是硬编码的，用户无法选择是否启用或调整压缩参数

**修复方案**:
- 在设置对话框中添加"图片"配置部分
- 添加压缩开关（默认开启）
- 添加压缩质量滑块（50-100%，默认 80%）
- 添加最大宽度设置（800-4096px，默认 1920px）
- 添加最大高度设置（600-4096px，默认 1080px）
- 压缩函数从 localStorage 读取用户配置

**修改文件**: 
- `app/ui/frontend/src/components/SettingsDialog.jsx`
- `app/ui/frontend/src/utils/imageCompressor.js`

**使用方法**:
1. 点击菜单栏"设置"按钮
2. 滚动到"图片"部分
3. 调整压缩参数
4. 点击"保存"

---

### 3. ✅ 预览区图片点击预览
**问题描述**: 图片预览功能只在图片管理对话框中可用，预览区的图片无法点击放大

**修复方案**:
- 在 App.jsx 中添加 ImagePreviewDialog 组件
- 监听预览区的点击事件
- 检测点击的是否为图片元素
- 点击图片时显示预览对话框

**修改文件**: `app/ui/frontend/src/App.jsx`

**使用方法**:
- 在右侧预览区直接点击任意图片即可放大查看

---

### 4. ✅ 批量管理按钮位置调整
**问题描述**: 批量管理按钮在图片库头部，不够显眼

**修复方案**:
- 将"批量管理"按钮移到对话框左下角
- 只在图片库标签页显示
- 进入批量模式后，操作按钮仍在头部显示
- 调整 footer 布局为 space-between

**修改文件**: 
- `app/ui/frontend/src/components/ImageManagerDialog.jsx`
- `app/ui/frontend/src/components/ImageManagerDialog.css`

**使用方法**:
1. 打开图片管理对话框
2. 切换到"图片库"标签
3. 点击左下角"批量管理"按钮
4. 选择图片后，使用头部的"全选"、"删除选中"、"取消"按钮

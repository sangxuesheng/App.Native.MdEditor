# Ctrl+V 粘贴图片测试说明（第四版）

## 🔧 本次修改

**问题**: window 级别的粘贴监听器没有触发

**解决方案**: 
- 移除 window 级别的粘贴监听器
- 改为直接在 Monaco Editor 的 DOM 节点上监听
- 在捕获阶段监听粘贴事件
- 添加详细的调试日志

## 📦 安装测试

```bash
cd /vol4/1000/开发文件夹/mac
appcenter-cli install-local
```

## 🧪 测试步骤

### 1. 打开浏览器控制台
- 按 F12 打开开发者工具
- 切换到 Console 标签

### 2. 检查监听器是否安装
- 刷新页面或打开编辑器
- 应该看到日志：`已在编辑器 DOM 上添加粘贴监听器`

### 3. 测试粘贴图片
- 截图（Windows: Win+Shift+S，Mac: Cmd+Shift+4）
- 在编辑器中点击，确保编辑器获得焦点
- 按 Ctrl+V（Mac: Cmd+V）

### 4. 查看控制台输出

**预期日志**:
```
已在编辑器 DOM 上添加粘贴监听器
编辑器粘贴事件触发 PasteEvent {...}
剪贴板项目数量: 1
项目 0 类型: image/png
发现图片，阻止默认行为
获取到图片文件: image.png 12345
正在压缩图片...
正在上传图片...
图片上传成功: 1234567890_abc123.png
```

**如果没有日志**:
- 说明粘贴事件还是没有触发
- 可能是 Monaco Editor 内部拦截了粘贴事件
- 需要使用其他方案（如工具栏按钮上传）

## 🔍 可能的情况

### 情况1: 看到完整日志
✅ 粘贴功能正常工作！

### 情况2: 只看到"已在编辑器 DOM 上添加粘贴监听器"
❌ 粘贴事件没有触发
- 可能是 Monaco Editor 内部拦截
- 需要使用 Monaco Editor 的 API 来处理粘贴

### 情况3: 什么日志都没有
❌ 监听器没有安装
- 可能是编辑器初始化问题
- 需要检查 handleEditorMount 是否被调用

## 💡 备选方案

如果粘贴功能还是不工作，我们可以：

1. **使用图片管理对话框上传**
   - 点击工具栏的图片按钮
   - 在对话框中上传图片
   - 这个功能是正常的

2. **添加工具栏快捷上传按钮**
   - 在工具栏添加"上传图片"按钮
   - 点击后直接选择文件上传

3. **使用拖拽上传**
   - 将图片文件拖到编辑器
   - 自动上传并插入

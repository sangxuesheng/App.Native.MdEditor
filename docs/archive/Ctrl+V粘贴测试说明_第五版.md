# Ctrl+V 粘贴图片测试说明（第五版）

## 🔧 本次修改

**问题**: Monaco Editor 拦截了编辑器内的粘贴事件

**解决方案**: 
- 在 **document 级别**监听粘贴事件（捕获阶段）
- 检查焦点是否在编辑器内
- 如果焦点在编辑器且剪贴板有图片，则拦截并上传
- 添加清理逻辑，组件卸载时移除监听器

**工作原理**:
```
用户按 Ctrl+V
  ↓
document 捕获阶段监听器触发
  ↓
检查焦点是否在编辑器内？
  ├─ 是 → 检查剪贴板是否有图片？
  │        ├─ 是 → 阻止默认行为，上传图片
  │        └─ 否 → 允许默认粘贴
  └─ 否 → 忽略（不处理）
```

## 📋 上传流程

当检测到图片粘贴时：

1. **压缩图片**（如果启用）
   - 读取设置中的压缩配置
   - 压缩质量、最大宽高等

2. **上传到服务器**
   - 调用 `/api/image/upload` API
   - 保存到本地图床：`/var/apps/App.Native.MdEditor2/shares/images/年/月/日/`

3. **插入 Markdown**
   - 格式：`![图片](/images/2026/03/01/xxx.jpg)`
   - 自动插入到光标位置

## 🎯 技术细节

### 为什么使用 document 级别监听？

1. **Monaco Editor 拦截**: Monaco 在编辑器内部处理粘贴，外部监听器无法触发
2. **捕获阶段**: 在事件到达 Monaco 之前拦截
3. **焦点检查**: 只处理编辑器内的粘贴，不影响其他输入框

### 事件流程

```
用户粘贴
  ↓
document (捕获阶段) ← 我们在这里拦截
  ↓
Monaco Editor (目标阶段) ← 如果我们不拦截，Monaco 会处理
  ↓
document (冒泡阶段)
```

### 代码关键点

```javascript
// 在 document 上监听，捕获阶段（第三个参数为 true）
document.addEventListener('paste', handleDocumentPaste, true)

// 检查焦点
const activeElement = document.activeElement
const editorDom = editor.getDomNode()
if (editorDom.contains(activeElement)) {
  // 焦点在编辑器内，处理粘贴
}
```

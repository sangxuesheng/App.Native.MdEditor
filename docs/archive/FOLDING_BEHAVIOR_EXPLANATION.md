# Markdown 折叠行为说明

## 当前折叠逻辑

### 折叠范围计算
当前的折叠提供器会为每个标题创建独立的折叠区域：

```
# 第一章 (折叠范围: 第1行到第15行)
内容...

## 1.1 小节 (折叠范围: 第4行到第8行)
小节内容

### 1.1.1 子小节 (折叠范围: 第7行到第8行)
子小节内容

## 1.2 小节 (折叠范围: 第10行到第15行)
另一个小节

# 第二章 (折叠范围: 第16行到最后)
```

### Monaco Editor 的折叠行为

Monaco Editor 默认支持两种折叠模式：

#### 1. 独立折叠（当前行为）
- 每个折叠区域独立
- 折叠一级标题时，只隐藏内容，但二级标题的折叠按钮仍然可见
- 需要分别折叠每个标题

**示例**:
```
▶ # 第一章 ...
```
折叠后，二级标题被隐藏，但如果展开，二级标题仍然可以独立折叠。

#### 2. 嵌套折叠（联动折叠）
- 折叠父级时，子级也被折叠
- 这是你期望的行为

## 问题分析

Monaco Editor 的折叠行为是**独立折叠**，不是**嵌套联动折叠**。这是 Monaco Editor 的设计决定，不是我们的代码问题。

## 解决方案

### 方案 A：接受当前行为（推荐）
这是 Monaco Editor 的标准行为，与 VS Code 一致。

**优点**:
- 符合 VS Code 的行为
- 用户可以灵活控制每个标题的折叠状态
- 不需要额外代码

**缺点**:
- 不是完全的"联动折叠"

### 方案 B：实现自定义联动折叠
需要监听折叠事件，手动折叠子级标题。

**优点**:
- 实现真正的联动折叠

**缺点**:
- 需要大量额外代码
- 可能影响性能
- 与 VS Code 行为不一致
- 用户失去灵活性

## 测试当前行为

### 测试文档
```markdown
# 第一章

这是第一章的内容

## 1.1 小节

小节内容

### 1.1.1 子小节

子小节内容

## 1.2 小节

另一个小节
```

### 测试步骤

1. **折叠一级标题**
   - 点击 "# 第一章" 左侧的 ▼
   - 结果: 整个第一章被折叠（包括所有子标题）
   - 显示: `▶ # 第一章 ...`

2. **展开一级标题**
   - 点击 "# 第一章" 左侧的 ▶
   - 结果: 第一章展开，显示所有子标题
   - 子标题仍然可以独立折叠

3. **折叠二级标题**
   - 点击 "## 1.1 小节" 左侧的 ▼
   - 结果: 该小节被折叠（包括三级标题）
   - 显示: `▶ ## 1.1 小节 ...`

### 预期行为

这是 Monaco Editor 的标准行为：
- ✅ 折叠父级标题会隐藏所有子级内容
- ✅ 展开父级标题会显示所有子级标题
- ✅ 子级标题可以独立折叠
- ⚠️ 折叠父级时，子级的折叠状态不会改变（这是设计如此）

## VS Code 对比

在 VS Code 中：
1. 折叠一级标题 → 所有内容被隐藏
2. 展开一级标题 → 所有子标题显示
3. 子标题可以独立折叠
4. 折叠父级不会改变子级的折叠状态

**我们的编辑器行为与 VS Code 完全一致！**

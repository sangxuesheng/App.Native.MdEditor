# 图片库功能实现说明

## 功能概述
实现了图片库标签页的实时加载功能，用户上传的图片和通过系统文件管理器添加的图片都会实时显示在图片库中。

## 实现内容

### 1. 后端API（app/server/server.js）
- **新增API**: `GET /api/image/list`
  - 递归扫描授权目录下的 `images/` 文件夹
  - 支持的图片格式：jpg, jpeg, png, gif, webp, svg
  - 返回图片列表，包含：url, filename, size, mtime, alt
  - 按修改时间倒序排列（最新的在前）

### 2. 前端组件（app/ui/frontend/src/components/ImageManagerDialog.jsx）
- **新增状态**:
  - `libraryImages`: 存储图片库列表
  - `loadingLibrary`: 加载状态标识

- **新增功能**:
  - `loadLibraryImages()`: 从后端加载图片列表
  - 自动加载：切换到图片库标签页时自动加载
  - 手动刷新：点击刷新按钮重新加载
  - 上传后刷新：上传成功后自动刷新图片库

- **UI改进**:
  - 显示图片数量
  - 刷新按钮（带加载动画）
  - 加载中状态提示
  - 空状态提示
  - 图片网格展示
  - 图片信息（文件名、大小）
  - 插入按钮

### 3. 样式（app/ui/frontend/src/components/ImageManagerDialog.css）
- 图片库布局样式
- 刷新按钮样式
- 加载动画
- 图片信息卡片样式

## 使用方式

### 上传图片
1. 在"上传图片"标签页上传图片
2. 图片会保存到授权目录的 `images/年/月/日/` 目录下
3. 上传成功后，如果当前在图片库标签页，会自动刷新

### 查看图片库
1. 切换到"图片库"标签页
2. 自动加载所有已上传的图片
3. 显示图片缩略图、文件名和大小
4. 点击"插入"按钮将图片插入到编辑器

### 手动刷新
1. 如果通过系统文件管理器添加了图片
2. 点击右上角的"刷新"按钮
3. 图片库会重新加载最新的图片列表

## 技术特点

1. **实时性**: 
   - 切换标签页自动加载
   - 上传后自动刷新
   - 支持手动刷新

2. **用户体验**:
   - 加载状态提示
   - 刷新按钮动画
   - 图片数量显示
   - 空状态友好提示

3. **性能优化**:
   - 使用useCallback避免重复创建函数
   - 按需加载（只在图片库标签页加载）
   - 图片按时间倒序排列

4. **兼容性**:
   - 支持多种图片格式
   - 递归扫描子目录
   - 路径安全验证

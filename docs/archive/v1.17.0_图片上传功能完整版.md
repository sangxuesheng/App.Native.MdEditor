# v1.17.0 图片上传功能完整版

## 🎉 新增功能

### 1. Ctrl+V 粘贴上传 ✅
**状态**: 已测试成功

**使用方法**:
1. 截图（Windows: Win+Shift+S，Linux: Shift+PrtSc）
2. 在编辑器中点击，确保焦点在编辑器内
3. 按 Ctrl+V
4. 自动上传并插入图片链接

**技术实现**:
- document 级别监听粘贴事件（捕获阶段）
- 检查焦点是否在编辑器内
- 拦截图片粘贴，阻止默认行为
- 自动压缩并上传

**使用方法**:
1. 点击工具栏的"上传图片"按钮（Upload 图标）

## 📋 完整上传流程

### 流程图
```
用户操作（粘贴/点击/拖拽）
  ↓
获取图片文件
  ↓
压缩图片（如果启用）
  ├─ 读取设置：压缩质量、最大宽高
  ├─ 使用 Canvas API 压缩
  └─ 保持原始文件名
  ↓
上传到服务器
  ├─ API: POST /api/image/upload
  ├─ 保存路径: /var/apps/App.Native.MdEditor2/shares/images/年/月/日/
  └─ 文件名: 时间戳_随机字符串.扩展名
  ↓
插入 Markdown
  ├─ 格式: ![图片](/images/2026/03/01/xxx.jpg)
  └─ 插入到光标位置
  ↓
完成
```

### 压缩设置
在"设置"对话框中配置：
- **启用压缩**: 开/关
- **压缩质量**: 0.1 - 1.0（默认 0.8）
- **最大宽度**: 像素（默认 1920）
- **最大高度**: 像素（默认 1080）

## 💡 使用建议

### 最佳实践

1. **截图后立即插入**: 使用 Ctrl+V 粘贴
2. **批量上传本地图片**: 使用工具栏按钮多选
3. **从文件管理器快速上传**: 使用拖拽
4. **大图片**: 启用压缩，设置合适的质量和尺寸
5. **小图标**: 关闭压缩，保持原图质量

### 快捷键

- **Ctrl+V**: 粘贴图片上传

## 🐛 故障排除

### 问题1: 粘贴不工作
**解决方法**:
- 确保焦点在编辑器内（点击编辑器）
- 检查浏览器控制台是否有错误
- 确认剪贴板中有图片

### 问题2: 拖拽没有反应
**解决方法**:
- 确保拖拽的是图片文件
- 检查文件格式是否支持
- 查看控制台日志

### 问题3: 上传失败
**解决方法**:
- 检查文件大小（限制 10MB）
- 检查网络连接
- 查看服务器日志

### 问题4: 图片太大
**解决方法**:
- 启用压缩功能
- 调整压缩质量（推荐 0.8）
- 设置最大尺寸（推荐 1920x1080）

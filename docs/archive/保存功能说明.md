# 保存功能说明

## 📋 功能区分

### 保存 vs 另存为

#### 保存（Save）
- **按钮**: 工具栏右侧的"保存"按钮
- **快捷键**: Ctrl+S / Cmd+S
- **行为**:
  - 有文件路径 → 直接保存到当前文件
  - 无文件路径 → 打开保存对话框，选择位置和文件名

#### 另存为（Save As）
- **按钮**: 菜单栏 → 文件 → 另存为
- **快捷键**: 无
- **行为**:
  - 始终打开另存为对话框
  - 可以保存到新位置
  - 可以修改文件名
  - 用于创建副本或重命名

---

## 🎯 使用场景

### 场景1: 新建文档
```
1. 打开编辑器（无文件）
2. 编辑内容
3. 点击"保存"按钮
   ↓
4. 打开保存对话框
5. 选择保存位置
6. 输入文件名
7. 点击确定
   ↓
8. 保存成功，currentPath 已设置
9. 后续编辑点击"保存"直接保存
```

### 场景2: 编辑已有文件
```
1. 打开一个 .md 文件
2. 编辑内容
3. 点击"保存"按钮
   ↓
4. 直接保存到当前文件
5. 无需选择位置
```

### 场景3: 另存为（创建副本）
```
1. 打开一个 .md 文件
2. 编辑内容
3. 菜单栏 → 文件 → 另存为
   ↓
4. 打开另存为对话框
5. 选择新位置
6. 输入新文件名
7. 点击确定
   ↓
8. 保存到新位置
9. currentPath 更新为新路径
```

## 🔧 技术实现

### 保存函数
```javascript
const handleSave = () => {
  if (currentPath) {
    // 如果有路径，直接保存
    autoSave.manualSave()
  } else {
    // 如果没有路径，打开保存对话框
    setShowSaveAsDialog(true)
  }
}
```

### 另存为函数
```javascript
const handleSaveAs = () => {
  // 始终打开另存为对话框
  setShowSaveAsDialog(true)
}
```

### 保存按钮
```jsx
<button 
  className="btn-primary" 
  onClick={handleSave}  // 调用保存函数
  disabled={false}       // 始终启用
>
  保存
</button>
```

## 💡 设计理念

### 保存（Save）
- **目的**: 快速保存当前工作
- **行为**: 智能判断
  - 已有路径 → 快速保存
  - 无路径 → 引导用户选择位置
- **用户体验**: 最小化操作步骤

### 另存为（Save As）
- **目的**: 创建副本或保存到新位置
- **行为**: 始终打开对话框
- **用户体验**: 明确的意图表达

## 🚀 后续扩展

### 另存为功能可以扩展为：

1. **另存为模板**
   - 保存当前文档为模板

2. **导出为其他格式**

   - 另存为新版本
   - 保留历史版本

# 保存对话框修复说明

## 🎯 修复目标

将新建文档保存时的对话框从"另存为"改为"保存"，包括：
1. 对话框标题显示"保存"
2. 底部按钮显示"保存"

---

## ✅ 修复内容

### 1. SaveAsDialog 组件支持两种模式

**新增参数**: `isSaveAs`
- `isSaveAs = true`: 另存为模式（默认）
- `isSaveAs = false`: 保存模式

**修改内容**:
```jsx
// 函数签名
const SaveAsDialog = ({ 
  onClose, 
  onConfirm, 
  rootDirs, 
  currentPath, 
  theme, 
  isSaveAs = true  // 新增参数
}) => {

// 对话框标题
<h2>{isSaveAs ? '另存为' : '保存'}</h2>

// 按钮文字
{loading ? '保存中...' : (isSaveAs ? '另存为' : '保存')}
```

### 2. App.jsx 智能判断模式

**新增状态**: `isSaveAsMode`
```javascript
const [isSaveAsMode, setIsSaveAsMode] = useState(true)
```

**保存按钮点击**:
```javascript
const handleSaveClick = () => {
  if (currentPath) {
    // 有路径：直接保存
    autoSave.manualSave()
  } else {
    // 无路径：打开保存对话框
    setIsSaveAsMode(false)  // 设置为保存模式
    setShowSaveAsDialog(true)
  }
}
```

**另存为按钮点击**:
```javascript
const handleSaveAs = () => {
  setIsSaveAsMode(true)  // 设置为另存为模式
  setShowSaveAsDialog(true)
}
```

### 3. 工具栏和保存按钮始终启用

**工具栏**:
```jsx
<EditorToolbar 
  onInsert={handleToolbarInsert}
  onImageUpload={handleImageUpload}
  onOpenImageManager={() => setShowImageManager(true)}
  onOpenTableInsert={() => setShowTableInsert(true)}
  disabled={false}  // 始终启用
/>
```

**保存按钮**:
```jsx
<button 
  className="btn-primary" 
  onClick={handleSaveClick} 
  disabled={false}  // 始终启用
>
  保存
</button>
```

## 🎨 使用场景

### 场景1: 新建文档保存
1. 打开编辑器（无文件）
2. 直接编辑内容
3. 点击"保存"按钮
4. 打开对话框：
   - 标题显示：**保存** ✅
   - 按钮显示：**保存** ✅
5. 选择位置和文件名
6. 点击"保存"按钮
7. 保存成功

### 场景2: 已有文件另存为
1. 打开一个 .md 文件
2. 编辑内容
3. 点击菜单"文件" → "另存为"
4. 打开对话框：
   - 标题显示：**另存为** ✅
   - 按钮显示：**另存为** ✅
5. 选择新位置和文件名
6. 点击"另存为"按钮
7. 保存成功

### 场景3: 已有文件保存
1. 打开一个 .md 文件
2. 编辑内容
3. 点击"保存"按钮
4. 直接保存到当前文件 ✅
5. 不打开对话框

## 🔧 技术实现

### 对话框模式切换
```
用户操作 → 设置模式 → 打开对话框 → 显示对应文字

保存按钮（无路径）
  ↓
setIsSaveAsMode(false)
  ↓
打开对话框
  ↓
显示"保存"

另存为按钮
  ↓
setIsSaveAsMode(true)
  ↓
打开对话框
  ↓
显示"另存为"
```

### 状态管理
- `showSaveAsDialog`: 控制对话框显示/隐藏
- `isSaveAsMode`: 控制对话框模式（保存/另存为）
- `currentPath`: 当前文件路径（用于判断是否需要打开对话框）

# 右键菜单剪贴板功能修复 - 最终版本

## 问题描述

### 原始问题
1. **开发模式**：剪切、复制可用，粘贴不可用
2. **生产模式**：剪切、复制、粘贴都不可用

### 第一次修复后的问题
使用 `editor.trigger()` 调用 Monaco Editor 命令时出现错误：
```
Error: [invokeFunction] unknown service 'productService'
```

## 根本原因

Monaco Editor 的 `editor.trigger()` 方法在某些环境下（特别是通过 CDN 加载或打包后）可能无法正确解析某些内部服务，导致剪贴板命令失败。

## 最终解决方案

使用 **document.execCommand** 作为主要方法，配合 **Clipboard API** 作为后备方案。

### 技术选择理由

1. **document.execCommand**
   - ✅ 浏览器原生支持，兼容性最好
   - ✅ 不依赖 Monaco Editor 内部服务
   - ✅ 在 HTTP 和 HTTPS 环境都能工作
   - ✅ 不需要用户授权
   - ⚠️ 已被标记为废弃，但仍广泛支持

2. **Clipboard API（后备）**
   - ✅ 现代浏览器标准 API
   - ✅ 功能更强大
   - ⚠️ 需要 HTTPS 环境
   - ⚠️ 需要用户授权

3. **Monaco Editor 命令（放弃）**
   - ❌ 依赖内部服务，可能失败
   - ❌ 在某些打包环境下不可用
   - ❌ 错误难以调试

## 实现代码

### 剪切操作
```javascript
case 'cut':
  if (contextMenu?.selectedText) {
    editor.focus()
    try {
      const success = document.execCommand('cut')
      if (success) {
        showToast('已剪切', 'success')
      } else {
        // 后备方案：手动实现
        const selection = editor.getSelection()
        const text = editor.getModel().getValueInRange(selection)
        navigator.clipboard.writeText(text).then(() => {
          editor.executeEdits('cut', [{
            range: selection,
            text: ''
          }])
          showToast('已剪切', 'success')
        }).catch(() => {
          showToast('剪切失败', 'error')
        })
      }
    } catch (error) {
      console.error('剪切失败:', error)
      showToast('剪切失败', 'error')
    }
  }
  break
```

### 复制操作
```javascript
case 'copy':
  if (contextMenu?.selectedText) {
    editor.focus()
    try {
      const success = document.execCommand('copy')
      if (success) {
        showToast('已复制', 'success')
      } else {
        // 后备方案：使用 Clipboard API
        navigator.clipboard.writeText(contextMenu.selectedText).then(() => {
          showToast('已复制', 'success')
        }).catch(() => {
          showToast('复制失败', 'error')
        })
      }
    } catch (error) {
      console.error('复制失败:', error)
      showToast('复制失败', 'error')
    }
  }
  break
```

### 粘贴操作
```javascript
case 'paste':
  editor.focus()
  try {
    const success = document.execCommand('paste')
    if (success) {
      showToast('已粘贴', 'success')
    } else {
      // 后备方案：使用 Clipboard API
      navigator.clipboard.readText().then(text => {
        const selection = editor.getSelection()
        editor.executeEdits('paste', [{
          range: selection,
          text: text
        }])
        showToast('已粘贴', 'success')
      }).catch(() => {
        showToast('粘贴失败，请使用 Ctrl+V', 'warning')
      })
    }
  } catch (error) {
    console.error('粘贴失败:', error)
    showToast('粘贴失败，请使用 Ctrl+V', 'warning')
  }
  break
```

## 工作流程

### 剪切/复制流程
```
1. 聚焦编辑器
2. 尝试 document.execCommand('cut'/'copy')
   ├─ 成功 → 显示成功提示
   └─ 失败 → 使用 Clipboard API
      ├─ 成功 → 显示成功提示
      └─ 失败 → 显示错误提示
```

### 粘贴流程
```
1. 聚焦编辑器
2. 尝试 document.execCommand('paste')
   ├─ 成功 → 显示成功提示
   └─ 失败 → 使用 Clipboard API
      ├─ 成功 → 手动插入文本
      └─ 失败 → 提示用户使用快捷键
```

## 用户体验改进

### 1. 友好的错误提示
- 成功操作：显示绿色成功提示
- 失败操作：显示红色错误提示
- 粘贴失败：提示用户使用快捷键（Ctrl+V）

### 2. 按钮状态
- 剪切/复制：仅在有选中文本时启用
- 粘贴：始终启用（因为无法可靠检测剪贴板内容）
- 删除：仅在有选中文本时启用

### 3. 操作反馈
所有操作都有即时的 Toast 通知，用户清楚知道操作是否成功。

## 已知限制

1. **移动浏览器**
   - 剪贴板操作在移动浏览器中可能受限
   - 建议移动端使用长按选择 + 系统菜单

2. **粘贴权限**
   - 某些浏览器可能阻止自动粘贴
   - 用户会看到提示使用 Ctrl+V

3. **execCommand 废弃**
   - `document.execCommand` 已被标记为废弃
   - 但仍被所有主流浏览器支持
   - 未来可能需要迁移到纯 Clipboard API

# 右键菜单功能修复总结 v1.0.2

## 🐛 修复的问题

### 问题 1：预览区图片缩放不生效
**现象**：在编辑区使用 HTML 语法写的图片，右键缩放后，预览区显示的图片大小没有变化。

**原因**：使用了 `style="zoom:50%;"` 属性，但 `zoom` 在某些浏览器或 Markdown 渲染环境中不生效。

**解决方案**：改用更通用的 `width` 属性。

**修改前**：
```html
<img src="..." alt="..." title="..." style="zoom:50%;" />
```

**修改后**：
```html
<img src="..." alt="..." title="..." width="50%" />
```

---

### 问题 2：预览区右键菜单显示不正确
**现象**：在预览区右键点击图片时，菜单显示了所有选项（包括编辑和格式化选项），但这些选项在预览区不应该显示。

**原因**：
1. `EditorContextMenu` 组件没有区分编辑区和预览区
2. 菜单逻辑没有根据上下文类型过滤选项

**解决方案**：
1. 添加 `contextType` 参数（'editor' | 'preview'）
2. 根据上下文类型显示不同的菜单选项

**位置**：`handleContextMenuAction` 函数中的 `case 'scale-image'`

```javascript
case 'scale-image':
  if (contextMenu?.selectedImage && data?.scale) {
    const image = contextMenu.selectedImage
    const zoomPercent = Math.round(data.scale * 100)
    
    // 使用 width 属性而不是 zoom

      range: image.range,
      text: newText
    }])

  }
  break
```

### 3. EditorContextMenu.jsx - 优化菜单生成逻辑

**位置**：`getMenuItems` 函数

```javascript
const getMenuItems = () => {
  const items = []

  // 如果选中了图片
  if (selectedImage) {
    // 添加图片相关选项
    items.push({ label: '上传图片', ... })
    items.push({ label: '缩放图片', ... })
    items.push({ label: '转换图片语法', ... })
    items.push({ label: '删除图片文件', ... })
    items.push({ divider: true })
    items.push({ label: '拷贝图片', ... })
    items.push({ label: '将图像另存为...', ... })
    
    // 如果是预览区，只显示图片选项，直接返回
    if (contextType === 'preview') {
      return items
    }
    
    items.push({ divider: true })
  }

  // 编辑工具栏（仅在编辑区显示）
  if (contextType === 'editor') {
    items.push({ type: 'toolbar', ... })
    items.push({ divider: true })
  }

  // 格式化选项（仅在编辑区显示）
  if (contextType === 'editor') {
    items.push({ label: '加粗', ... })
    items.push({ label: '斜体', ... })
    // ... 其他格式化选项
    items.push({ label: '段落', ... })
    items.push({ label: '插入', ... })
  } else {
    // 预览区：只显示复制选项
    if (selectedText) {
      items.push({ label: '复制', ... })
    }
  }

  return items
}
```

## 🎨 技术细节

### width vs zoom

| 属性 | 兼容性 | 效果 | 推荐 |
|------|--------|------|------|
| `zoom` | 部分浏览器支持 | 缩放整个元素 | ❌ |
| `width` | 所有浏览器支持 | 调整宽度，高度自适应 | ✅ |

### 上下文类型判断

```javascript
// 编辑区
contextType === 'editor'
  → 显示完整菜单（图片 + 编辑 + 格式化）

// 预览区
contextType === 'preview'
  → 只显示图片选项或复制选项
```

---

## ⚠️ 注意事项

1. **width 属性的行为**
   - `width="50%"` 表示相对于父容器的 50%
   - 如果父容器宽度变化，图片大小也会变化
   - 高度会自动按比例调整

2. **预览区菜单限制**
   - 预览区不支持编辑操作（剪切、粘贴、删除）
   - 预览区不支持格式化操作（加粗、斜体等）
   - 预览区只支持复制和图片操作

3. **图片查找逻辑**
   - 通过 `src` 属性在编辑器中查找对应语法
   - 支持完整 URL 和相对路径匹配
   - 如果有多个相同 src 的图片，会匹配第一个

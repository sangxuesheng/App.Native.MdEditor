## 🐛 问题 1：预览区图片不按比例缩放

### 问题描述
使用 `style="zoom:50%;"` 时，预览区的图片没有按比例缩放，显示大小相同。

### 原因分析
`zoom` CSS 属性在某些浏览器或渲染环境中不被支持或不生效。

### 解决方案
改用标准的 `width` 和 `height` 样式：

```html
<!-- 之前 -->
<img src="..." alt="..." style="zoom:50%;" />

<!-- 修复后 -->
<img src="..." alt="..." style="width:50%;height:auto;" />
```

### 代码修改
```javascript
case 'scale-image':
  const widthPercent = Math.round(data.scale * 100)
  const titleAttr = image.title ? ` title="${image.title}"` : ''
  
  // 使用 width 和 height 样式
  newText = `<img src="${image.src}" alt="${image.alt}"${titleAttr} style="width:${widthPercent}%;height:auto;" />`
```

## 🐛 问题 2：预览区 Markdown 图片无法缩放

### 问题描述
在预览区右键点击由 Markdown 语法渲染的图片时，无法进行缩放设置。

### 原因分析
1. 预览区的图片是由 Markdown 渲染生成的 HTML
2. 图片的 `src` 可能是完整 URL，而编辑区可能是相对路径
3. 原有的匹配逻辑不够全面，无法匹配所有情况

### 解决方案
增强 `findImageInEditor()` 函数的匹配逻辑，支持多种匹配方式：

1. **完全匹配**：`matchedSrc === imageSrc`
2. **路径匹配**：`matchedSrc === srcPath`
3. **包含匹配**：`imageSrc.includes(matchedSrc)`
4. **文件名匹配**：`matchedSrc.includes(fileName)`
5. **结尾匹配**：`imageSrc.endsWith(matchedSrc)`
6. **URL 文件名匹配**：比较 URL 的最后一部分

### 代码修改
```javascript
const findImageInEditor = useCallback((imageSrc) => {
  // 提取路径和文件名
  const srcPath = imageSrc.replace(/^https?:\/\/[^\/]+/, '')
  const fileName = imageSrc.split('/').pop()
  
  // 多种匹配方式
  const isMatch = 
    matchedSrc === imageSrc ||                    // 完全匹配
    matchedSrc === srcPath ||                     // 路径匹配
    imageSrc.includes(matchedSrc) ||              // 包含匹配
    matchedSrc.includes(fileName) ||              // 文件名匹配
    imageSrc.endsWith(matchedSrc) ||              // 结尾匹配
    (matchedSrc.startsWith('http') && 
     imageSrc.includes(matchedSrc.split('/').pop())) // URL 文件名匹配
  
  if (isMatch) {
    // 返回图片信息和位置
  }
}, [])
```

### 添加调试日志
为了方便调试，添加了 console.log 输出：
- 查找图片时输出：`imageSrc`, `srcPath`, `fileName`
- 找到匹配时输出：`lineNumber`, `matchedSrc`
- 未找到时输出：`未找到匹配的图片`

## 📊 匹配逻辑对比

### 修复前
```javascript
// 只有 3 种匹配方式
if (matchedSrc === imageSrc || 
    matchedSrc === srcPath || 
    imageSrc.includes(matchedSrc)) {
  // 匹配成功
}
```

**问题**：
- ❌ 无法匹配文件名相同但路径不同的情况
- ❌ 无法匹配 URL 编码不同的情况
- ❌ 无法匹配域名不同但路径相同的情况

### 修复后
```javascript
// 6 种匹配方式
const isMatch = 
  matchedSrc === imageSrc ||                    // 完全匹配
  matchedSrc === srcPath ||                     // 路径匹配
  imageSrc.includes(matchedSrc) ||              // 包含匹配
  matchedSrc.includes(fileName) ||              // 文件名匹配
  imageSrc.endsWith(matchedSrc) ||              // 结尾匹配
  (matchedSrc.startsWith('http') && 
   imageSrc.includes(matchedSrc.split('/').pop())) // URL 文件名匹配
```

**优势**：
- ✅ 支持更多 URL 格式
- ✅ 支持文件名匹配
- ✅ 支持结尾匹配
- ✅ 更强的容错性

## 📈 性能优化

### 当前性能
- 遍历所有行：O(n)，n 为文档行数
- 正则匹配：O(m)，m 为每行字符数
- 总复杂度：O(n × m)

### 适用场景
- ✅ 小型文档（< 1000 行）：性能优秀
- ✅ 中型文档（1000-5000 行）：性能良好
- ⚠️ 大型文档（> 5000 行）：可能有轻微延迟

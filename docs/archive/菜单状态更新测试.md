# 菜单状态更新测试

## 测试步骤

### 1. 刷新页面
按 Ctrl+R 或 F5 刷新页面

### 2. 在预览区右键点击 50% 的图片

### 3. 查看控制台输出

应该看到：
```
预览区右键菜单 - 检测到的图片: {scale: 0.5, ...}
最终的 selectedImage: {scale: 0.5, ...}
```

### 4. 如果没有看到日志

在控制台执行：
```javascript
// 手动测试
const img = document.querySelector('.markdown-body img')
const style = img.getAttribute('style')
console.log('style:', style)

const widthMatch = style?.match(/width:\s*(\d+)%/)
console.log('widthMatch:', widthMatch)

const scale = widthMatch ? parseInt(widthMatch[1]) / 100 : 1
console.log('scale:', scale)
```

### 5. 检查菜单组件接收到的 props

在 EditorContextMenu.jsx 中，selectedImage.scale 应该是 0.5

如果 scale 是 0.5 但菜单还是显示 100%，说明菜单组件的比较逻辑有问题。

### 6. 检查比较逻辑

EditorContextMenu.jsx 第 295 行：
```javascript
className={`editor-submenu-item with-check ${selectedImage?.scale === option.value ? 'active' : ''}`}
```

这里比较的是：
- selectedImage.scale (应该是 0.5)
- option.value (50% 对应 0.5)

如果相等，应该添加 'active' 类。

### 7. 手动检查菜单

在控制台执行：
```javascript
// 检查菜单项
const menuItems = document.querySelectorAll('.editor-submenu-item')
menuItems.forEach((item, i) => {
  console.log(`菜单项 ${i}:`, {
    text: item.textContent,
    hasActive: item.classList.contains('active'),
    classes: item.className
  })
})
```

应该看到 "50%" 的菜单项有 'active' 类。

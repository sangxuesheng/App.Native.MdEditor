> 基于需求文档《复制使用的文件可以删除》制定，**核心原则：不影响现有 .md 功能**

**执行状态**：✅ 已完成（详见 `全格式支持-开发完成总结.md`）

## 一、现状与影响范围

### 1.1 当前实现要点

| 模块 | 现状 | 需改动 |
|------|------|--------|
| **后端 `/api/files`** | 仅返回 `.md` 文件（`!entry.name.endsWith('.md')` 过滤） | 改为返回所有文件，可选增加扩展信息 |
| **后端 `/api/file`** | 使用 `utf8` 读取，任意路径可读 | 需支持 binary/hex 模式、PDF 纯文本提取 |
| **FileTree** | 依赖 `/api/files`，只展示 .md | 随 API 变更自动展示全部文件 |
| **FileBrowser** | 用于 SaveAs/NewFile，同样依赖 `/api/files` | 同上 |
| **loadFile** | 直接 `fetch` 文本，`setContent` | 需按格式分支：文本/图片/二进制/PDF |
| **布局 layout** | `vertical` / `editor-only` / `preview-only` | 需增加「按格式自动切换」逻辑 |
| **Monaco 编辑器** | 固定 `markdown` 语言模式 | 需按扩展名切换 `language` |
| **预览区** | 仅 Markdown 富文本渲染 | 需支持：纯文本、图片、Hex、「不支持预览」 |

### 1.2 兼容性保障

- **MD 格式**：保持现有行为，布局「保留用户当前」，预览照常渲染
- **现有 API 契约**：`/api/file` 返回 `{ ok, content }` 保持不变，新增可选字段（如 `encoding`、`format`）
- **布局持久化**：`persistSetting('layout')` 继续生效，仅「打开新文件时」按格式覆盖默认值

### 阶段 1：后端 API 扩展（2 天）✅ 已完成

#### 1.1 `/api/files` 改造

**文件**：`app/server/server.js`

- **移除**：`entry.isFile() && !entry.name.endsWith('.md')` 过滤
- **新增**：返回 `stat` 信息（size、mtime），供前端着色、排序
- **可选**：增加 `?filter=all|supported` 查询参数，默认 `all`

```javascript
// 改造后：展示所有文件
.filter(entry => {
  if (entry.name.startsWith('.')) return false;
  // 移除：if (entry.isFile() && !entry.name.endsWith('.md')) return false;
  if (entry.isDirectory() && (entry.name === 'images' || entry.name === 'history')) return false;
  return true;
})
.map(entry => {
  const item = { name: entry.name, path: path.join(...), type: 'file'|'directory', isRoot: false };
  if (entry.isFile()) {
    try {
      const stat = fs.statSync(path.join(safePath, entry.name));
      item.size = stat.size;
      item.mtime = stat.mtimeMs;
    } catch (_) {}
  }
  return item;
})
```

#### 1.2 `/api/file` 扩展

- **新增查询参数**：`?path=xxx&mode=text|binary|hex`
- **text 模式**（默认）：保持现有 `utf8` 读取，与现有逻辑兼容
- **binary 模式**：`fs.readFile` 无 encoding，返回 Base64 或 ArrayBuffer（需与前端约定）
- **hex 模式**：读取 binary 后转为 hex 字符串返回

**响应扩展**（向后兼容）：

```javascript
{ ok: true, path, content, encoding?: 'utf8'|'base64'|'hex', format?: 'md'|'text'|'image'|'unsupported' }
```

#### 1.3 新增 `/api/file/pdf-extract`（可选独立接口）

- 使用 `pdf-parse` 或类似库提取纯文本
- 请求：`POST { path }`，响应：`{ ok, text }`
- 若提取失败，返回 `{ ok: false, text: '' }`，前端按纯文本兜底

**依赖**：需在 `app/server/package.json` 增加 `pdf-parse`（或轻量替代）

### 阶段 3：打开逻辑与自动布局（2.5 天）✅ 已完成

#### 3.1 loadFile 重构

**文件**：`app/ui/frontend/src/App.jsx`

**新流程**：

1. 根据 `getFormatFromPath(path)` 判断格式
2. **MD**：保持现有逻辑（`fetch` text → `setContent`），**不改变布局**
3. **文本类**：`fetch` text → `setContent`，**自动 `setLayout('editor-only')`**，显示 3 秒 toast
4. **图片类**：`fetch` binary/base64 → 存为 `imageDataRef`，**自动 `setLayout('preview-only')`**，预览区显示图片
5. **非支持格式**：弹窗「纯文本/Hex/取消」→ 若选纯文本，按 text 读取（可能乱码）；若选 Hex，按 hex 读取
6. **异常**：`setLayout('editor-only')`，toast 提示

**状态扩展**：

```javascript
const [currentFileFormat, setCurrentFileFormat] = useState(null)  // 'md'|'text'|'image'|'unsupported'
const [currentFileEncoding, setCurrentFileEncoding] = useState('utf8')  // 'utf8'|'hex'
const imageDataRef = useRef(null)  // 图片 base64/dataURL
```

#### 3.2 布局自动切换规则

| 格式 | 默认布局 | Toast |
|------|----------|-------|
| MD | 保留当前 | 无 |
| 文本类 | editor-only | 「文本类文件默认仅编辑模式打开，可手动切换双栏」 |
| 图片类 | preview-only | 「图片类文件默认仅预览模式打开，可手动切换双栏」 |
| 非支持 | editor-only | 「非支持格式默认仅编辑模式打开，可手动切换双栏」 |
| 异常 | editor-only | 「文件读取异常，默认仅编辑模式打开」 |

**实现要点**：在 `loadFile` 成功分支内，根据 `currentFileFormat` 调用 `setLayout`，并 `showToast`（3 秒自动消失）。**不**修改 `persistSetting` 的写入时机，仅影响「本次打开」的默认布局。

#### 3.3 新文件不继承上一文件布局

- 每次 `loadFile` 时，按格式重新应用默认布局
- 用户手动切换后，`layout` 状态保持，直到关闭文件或打开新文件

### 阶段 4：编辑器与预览区适配（3 天）✅ 已完成

#### 4.1 Monaco 语言模式

- 从 `getLanguageFromPath(currentPath)` 获取语言
- 在 `loadFile` 成功后、`setContent` 之后，调用 `editorRef.current?.getModel()?.setLanguage(lang)`
- 或通过 `useEffect` 监听 `currentPath` 变化，更新 Monaco 的 `language` 选项

#### 4.2 预览区多模式

**当前**：仅 `renderMarkdown` 渲染 HTML。

**扩展**：

- **MD**：继续使用 `renderMarkdown`，无变化
- **文本类**：双栏/仅预览时，预览区显示 `<pre>` 包裹的纯文本，可选语法高亮（如 highlight.js 或复用 Monaco 的 tokenize）
- **图片类**：`<img src={imageDataRef.current} />`，支持点击放大（简单 modal）
- **非支持**：显示「此格式不支持预览」

**实现**：在 `App.jsx` 的预览区渲染逻辑中，根据 `currentFileFormat` 分支：

```jsx
{currentFileFormat === 'md' && renderMarkdown(...)}
{currentFileFormat === 'text' && <pre className="preview-plaintext">{content}</pre>}
{currentFileFormat === 'image' && <img src={imageDataRef.current} alt="预览" />}
{currentFileFormat === 'unsupported' && <div>此格式不支持预览</div>}
```

#### 4.3 图片 Hex 编辑

- 图片类 + 仅编辑/双栏时，编辑区显示 Hex 字符串（只读或可编辑，按需求）
- 使用 `monaco.editor.setModelLanguage(model, 'plaintext')`，或自定义简单 textarea

### 阶段 5：保存逻辑（2 天）✅ 已完成

#### 5.1 保存分支

| 格式 | 行为 |
|------|------|
| MD | 直接保存，无提示 |
| 文本类（非 .pdf） | 直接保存 |
| .pdf | 弹窗：「仅保存提取的纯文本，原 PDF 将被覆盖」→ 确认/另存为 .md/取消 |
| 图片类 | 弹窗：「修改图片二进制可能导致损坏」→ 确认/另存为 .md/取消 |
| 非支持 | 弹窗：「修改该格式可能导致损坏」→ 确认保存/另存为 .md/取消 |

#### 5.2 后端写入

- **文本**：保持 `POST /api/file`，`{ path, content }`
- **二进制**：新增 `POST /api/file/binary`，`{ path, content: base64 }`，后端 `Buffer.from(content, 'base64')` 写入
- **Hex**：前端 hex → binary → base64，调用 binary 接口

#### 5.3 保存前校验

- 非支持格式、图片、PDF：在 `saveFile` 内部根据 `currentFileFormat` 先 `requestConfirm`，再调用保存 API

## 五、风险与缓解

| 风险 | 缓解 |
|------|------|
| 二进制文件用 utf8 读取导致乱码/崩溃 | 先判断格式，非文本用 binary/hex 接口 |
| PDF 依赖增加包体积 | 使用轻量库，或延迟加载 |
| 大文件内存占用 | 大文件提示确认，可后续做流式/分块 |
| Monaco 语言切换闪烁 | 在 `setContent` 之后同一帧更新 language |

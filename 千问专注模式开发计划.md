# 专注模式开发计划（优化版）

## 1. 架构设计原则

### 1.1 状态模型收敛
**问题识别**：避免 `layout + isFocusMode` 双状态冲突  
**解决方案**：采用单一 `viewMode` 状态模型
- `split`：左右布局（对应现有 `vertical`）
- `focus-edit`：专注编辑模式（对应现有 `editor-only`）  
- `focus-preview`：专注预览模式（对应现有 `preview-only`）

**向后兼容策略**：
- 现有 `layout` 状态自动映射到新 `viewMode`
- 平滑迁移，不影响现有用户数据和功能

### 1.2 组件化条件渲染
**问题识别**：避免 `.focus-mode { display:none !important }` 的维护成本  
**解决方案**：通过 `LayoutRouter` 实现组件级条件渲染
- `SplitLayout`：左右布局组件
- `FocusEditLayout`：专注编辑布局组件  
- `FocusPreviewLayout`：专注预览布局组件
- **优势**：不渲染不需要的DOM节点，提升性能和可维护性

### 1.3 渐进式演进策略
- **核心路径**：CSS/布局层专注模式（功能等价可用）
- **增强体验**：Fullscreen API作为可选增强（检测支持性后启用）
- **特性开关**：默认关闭，支持灰度发布和一键回退

### 2.2 状态管理 Hook
```javascript
// useViewModeState.js
const useViewModeState = () => {
  // 单一状态源
  const [viewMode, setViewMode] = useState(getInitialViewMode);
  
  // 特性开关检查
  const isFocusModeEnabled = useCallback(() => {
    return getFeatureFlag('focusModeEnabled', false);
  }, []);
  
  // 安全切换
  const switchViewMode = useCallback((newMode) => {
    if (!isFocusModeEnabled() && newMode !== 'split') {
      return; // 自动降级到split模式
    }
    setViewMode(newMode);
    persistSetting('viewMode', newMode);
  }, [isFocusModeEnabled]);
  
  return { viewMode, switchViewMode, isFocusModeEnabled };
};
```

### 2.3 布局路由组件
```javascript
// LayoutRouter.jsx
const LayoutRouter = ({ viewMode, ...props }) => {
  // 状态保持：保存和恢复光标/滚动位置
  const { saveEditorState, restoreEditorState } = useEditorStatePersistence();
  
  useEffect(() => {
    saveEditorState();
    return () => restoreEditorState();
  }, [viewMode]);
  
  switch (viewMode) {
    case 'split':
      return <SplitLayout {...props} />;
    case 'focus-edit':
      return <FocusEditLayout {...props} />;
    case 'focus-preview':
      return <FocusPreviewLayout {...props} />;
    default:
      return <SplitLayout {...props} />; // 默认降级
  }
};
```

### 2.4 大屏专注模式专项优化
**问题识别**：大屏用户在专注模式下光标可能位于屏幕边缘，影响专注度和编辑体验

**解决方案**：智能光标居中 + 行宽限制

#### 2.4.1 智能光标居中
```javascript
// useLargeScreenOptimization.js
const useLargeScreenOptimization = (editorRef, isEnabled, viewMode) => {
  const config = useMemo(() => ({
    cursorAutoCenter: true,
    maxWidth: 800,
    minScreenHeight: 800,
    debounceDelay: 100
  }), []);
  
  useEffect(() => {
    if (!isEnabled || viewMode !== 'focus-edit' || !editorRef.current) {
      return;
    }
    
    const editor = editorRef.current;
    const shouldEnableCentering = window.innerHeight > config.minScreenHeight;
    
    if (shouldEnableCentering) {
      const debouncedCenter = debounce((position) => {
        try {
          editor.revealLineInCenter(position.lineNumber);
        } catch (error) {
          console.warn('Failed to center cursor:', error);
        }
      }, config.debounceDelay);
      
      const disposable = editor.onDidChangeCursorPosition((e) => {
        debouncedCenter(e.position);
      });
      
      return () => {
        disposable.dispose();
        debouncedCenter.cancel();
      };
    }
  }, [isEnabled, viewMode, editorRef, config]);
  
  return config;
};
```

#### 2.4.3 设备差异化策略
- **小屏（< 768px）**：禁用自动居中，保持默认行为
- **中屏（768-1200px）**：仅提供手动居中快捷键（`Ctrl/Cmd + L`）
- **大屏（> 1200px）**：自动居中，但提供用户设置选项控制

```javascript
// useFocusModeEnhancement.js
const useFocusModeEnhancement = (isEnabled) => {
  const requestFullscreenEnhancement = async () => {
    // 检测Fullscreen API支持
    if (!document.documentElement.requestFullscreen) {
      return false; // 不支持，使用CSS布局
    }
    
    try {
      await document.documentElement.requestFullscreen();
      return true;
    } catch (error) {
      console.warn('Fullscreen API failed, falling back to CSS layout');
      return false;
    }
  };
  
  return { requestFullscreenEnhancement };
};
```

## 4. 性能保障措施

### 4.1 现实化性能指标
**相对基线指标（非绝对值）**：
- **切换性能**：专注模式切换不劣于现有布局切换性能
- **流畅度**：移动端P95帧率 ≥ 55 FPS（基准60 FPS）
- **内存占用**：相对基线增长 ≤ 10%
- **启动延迟**：专注模式进入延迟 ≤ 现有模式切换延迟

### 4.2 大屏优化性能保障
- **防抖处理**：光标居中操作添加100ms防抖
- **条件启用**：仅在大屏设备上启用居中功能
- **资源清理**：组件卸载时清理所有事件监听器
- **错误处理**：Monaco API调用添加try-catch保护

### 4.3 优化策略
- **资源隔离**：专注模式下暂停非必要后台任务
- **DOM优化**：只渲染当前需要的布局组件
- **动画优化**：禁用非必要CSS动画，减少重绘
- **内存管理**：组件卸载时清理所有引用和定时器

## 6. 架构约束与实施规范

### 6.1 App.jsx修改约束
**核心原则**："不新增业务逻辑，只允许接线改动"
**允许的操作**：
- 导入新Hook：`import { useViewModeState } from './hooks/useViewModeState'`
- 调用Hook获取状态：`const { viewMode, switchViewMode } = useViewModeState()`
- 条件渲染LayoutRouter：`<LayoutRouter viewMode={viewMode} {...props} />`
- 传递必要props

**禁止的操作**：
- ❌ 任何业务逻辑代码
- ❌ 状态管理实现
- ❌ 副作用处理
- ❌ 复杂条件分支

### 6.2 特性开关与灰度策略
```javascript
// 特性配置
const FOCUS_MODE_FEATURE_CONFIG = {
  enabled: false,           // 全局开关，默认关闭
  desktopEnabled: false,    // 桌面端灰度
  mobileEnabled: false,     // 移动端灰度  
  experimentalGestures: false, // 实验性手势，默认关闭

// 一键回退机制
const ensureCompatibleViewMode = (requestedMode, featureConfig) => {
  if (!featureConfig.enabled) return 'split';
  if (!featureConfig.desktopEnabled && isDesktop()) return 'split';
  if (!featureConfig.mobileEnabled && isMobile()) return 'split';
  return requestedMode;
};
```

### 8.1 技术风险

### 8.3 性能风险
- **内存泄漏**：严格遵循资源清理规范，组件卸载时清理所有引用
- **性能退化**：以现有功能为基准，确保不劣化用户体验
- **高频输入卡顿**：通过防抖和条件启用策略控制性能影响

本开发计划严格遵循项目架构规范，采用独立组件模式实现完全解耦，确保最小化核心文件修改，并提供了完整的回滚兼容方案和渐进式上线策略。特别针对大屏用户优化了专注模式体验，通过智能光标居中和行宽限制提升编辑效率，同时确保不影响其他设备的用户体验。

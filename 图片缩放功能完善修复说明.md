## 🐛 问题 1：预览区图片不按比例缩放

### 问题描述
编辑区已经设置了 `style="width:33%;height:auto;"`，但预览区的图片显示大小相同，没有按比例缩放。

### 原因分析
CSS 中的 `.markdown-body .image-figure img` 设置了 `max-width: 100%`，这个样式**覆盖了**内联样式的 `width` 属性。

```css
/* 问题代码 */
.markdown-body .image-figure img {
  max-width: 100%;  /* 这会限制所有图片最大宽度为 100% */
  height: auto;
}
```

### 解决方案
添加一个更高优先级的 CSS 规则，让带有 `width` 内联样式的图片不受 `max-width` 限制：

```css
/* 修复后 */
.markdown-body .image-figure img {
  max-width: 100%;
  height: auto;
  display: block;
  margin: 0 auto;
}

/* 支持内联样式的图片缩放 */
.markdown-body img[style*="width"] {
  max-width: none !important;
}
```

**关键点**：
- 使用属性选择器 `img[style*="width"]` 匹配所有带有 width 样式的图片
- 使用 `!important` 确保优先级最高
- 设置 `max-width: none` 移除宽度限制

## 🐛 问题 2：连续两张相同图片时缩放错误

### 问题描述
当编辑区有两张相同 src 的 HTML 图片时：
```html
<img src="https://example.com/image.jpg" alt="图1" />
<img src="https://example.com/image.jpg" alt="图2" />
```

在预览区右键点击第二张图片并缩放时，实际修改的是第一张图片。

### 原因分析
`findImageInEditor()` 函数在找到第一个匹配的图片后就立即返回，没有考虑可能有多个相同 src 的图片。

```javascript
// 问题代码
if (isMatch) {
  return { ... }  // 找到第一个就返回
}
```

### 解决方案

#### 1. 在预览区检测时记录图片索引
```javascript
const detectPreviewImage = useCallback((target) => {
  const imgElement = target.closest('img') || (target.tagName === 'IMG' ? target : null)
  
  if (imgElement && previewRef.current) {
    // 获取预览区所有图片
    const allImages = Array.from(previewRef.current.querySelectorAll('img'))
    const imageIndex = allImages.indexOf(imgElement)  // 记录索引
    
    return {
      alt: imgElement.alt || '',
      src: imgElement.src,
      imageIndex: imageIndex  // 添加索引信息
    }
  }
}, [])
```

#### 2. 修改查找函数支持索引参数
```javascript
const findImageInEditor = useCallback((imageSrc, targetIndex = 0) => {
  let matchCount = 0  // 记录匹配次数
  
  for (let lineNumber = 1; lineNumber <= lineCount; lineNumber++) {
    // ... 匹配逻辑 ...
    
    if (isMatch) {
      // 检查是否是目标索引的匹配
      if (matchCount === targetIndex) {
        return { ... }  // 返回第 N 个匹配
      }
      matchCount++  // 继续查找
    }
  }
}, [])
```

#### 3. 传递索引参数
```javascript
const handlePreviewContextMenu = useCallback((e) => {
  const previewImage = detectPreviewImage(e.target)
  
  if (previewImage) {
    // 使用图片索引来查找对应的编辑区图片
    selectedImage = findImageInEditor(previewImage.src, previewImage.imageIndex)
  }
}, [detectPreviewImage, findImageInEditor])
```

## 🎯 工作原理

### 完整流程

```
用户在预览区右键点击第二张图片
    ↓
detectPreviewImage() 检测图片
    ↓
获取预览区所有图片：[img1, img2, img3]
    ↓
计算当前图片索引：imageIndex = 1 (第二张)
    ↓
findImageInEditor(src, 1) 查找编辑区
    ↓
遍历编辑区，找到所有匹配的图片
    ↓
matchCount = 0: 找到第一张，跳过
matchCount = 1: 找到第二张，返回！✅
    ↓
用户选择缩放比例
    ↓
修改编辑区第二张图片的语法
    ↓
预览区第二张图片按比例缩放 ✨
```

## 🎨 CSS 优先级说明

### 优先级顺序（从低到高）

1. **默认样式**：`max-width: 100%`
2. **属性选择器 + !important**：`img[style*="width"] { max-width: none !important; }`

### 为什么使用 !important

虽然通常不推荐使用 `!important`，但在这种情况下是必要的：

1. **覆盖第三方 CSS**：github-markdown-css 可能有很高优先级的样式
2. **确保内联样式生效**：内联样式应该是最终决定因素
3. **避免复杂的选择器**：使用 `!important` 比写很长的选择器更清晰

### 当前性能
- 图片索引计算：O(n)，n 为预览区图片数量
- 编辑区查找：O(m)，m 为文档行数
- 总复杂度：O(n + m)

### 适用场景
- ✅ 小型文档（< 100 张图片）：性能优秀
- ✅ 中型文档（100-500 张图片）：性能良好

# 导出公众号格式功能修复

## 问题描述

导出公众号格式时出现错误：
```
导出失败: Cannot read properties of undefined (reading 'writeText')
```

## 问题原因

原代码直接使用 `navigator.clipboard.writeText()`，但在某些环境下：
1. `navigator.clipboard` 可能为 `undefined`
2. 需要 HTTPS 环境或 localhost
3. 需要用户授权
4. 某些浏览器不支持

```javascript
// 原代码 - 问题所在
case 'wechat':
  const wechatHtml = previewRef.current?.innerHTML || ''
  navigator.clipboard.writeText(wechatHtml).then(() => {
    // 成功处理
  }).catch(() => {
    // 错误处理不完善
  })
  break
```

## 解决方案

添加完整的降级方案，确保在任何环境下都能工作：

### 1. 添加降级辅助函数

```javascript
// 剪贴板降级方案：使用 textarea + execCommand
const copyToClipboardFallback = (text) => {
  const textarea = document.createElement('textarea')
  textarea.value = text
  textarea.style.position = 'fixed'
  textarea.style.opacity = '0'
  document.body.appendChild(textarea)
  textarea.select()
  
  try {
    const success = document.execCommand('copy')
    if (success) {
      setStatus('已复制公众号格式到剪贴板')
      setStatusType('success')
      setTimeout(() => {
        setStatus('就绪')
        setStatusType('normal')
      }, 2000)
    } else {
      setStatus('复制失败，请手动复制')
      setStatusType('error')
    }
  } catch (err) {
    console.error('execCommand 失败:', err)
    setStatus('复制失败，请手动复制')
    setStatusType('error')
  } finally {
    document.body.removeChild(textarea)
  }
}
```

### 2. 修改导出逻辑

```javascript
case 'wechat':
  // 公众号格式 - 复制到剪贴板
  const wechatHtml = previewRef.current?.innerHTML || ''
  
  // 优先使用 Clipboard API
  if (navigator.clipboard && navigator.clipboard.writeText) {
    navigator.clipboard.writeText(wechatHtml).then(() => {
      setStatus('已复制公众号格式到剪贴板')
      setStatusType('success')
      setTimeout(() => {
        setStatus('就绪')
        setStatusType('normal')
      }, 2000)
    }).catch((error) => {
      console.error('Clipboard API 失败:', error)
      // 降级方案：使用 textarea
      copyToClipboardFallback(wechatHtml)
    })
  } else {
    // 降级方案：使用 textarea
    copyToClipboardFallback(wechatHtml)
  }
  break
```

## 降级方案原理

### Textarea + execCommand 方法

1. **创建隐藏的 textarea**
   ```javascript
   const textarea = document.createElement('textarea')
   textarea.value = text
   textarea.style.position = 'fixed'
   textarea.style.opacity = '0'
   ```

2. **添加到 DOM 并选中**
   ```javascript
   document.body.appendChild(textarea)
   textarea.select()
   ```

3. **执行复制命令**
   ```javascript
   document.execCommand('copy')
   ```

4. **清理**
   ```javascript
   document.body.removeChild(textarea)
   ```

### 为什么这个方法更可靠

- ✅ 不需要 HTTPS
- ✅ 不需要用户授权
- ✅ 兼容所有浏览器
- ✅ 在 HTTP 环境下也能工作

## 使用说明

### 导出到微信公众号

1. 在 Markdown 编辑器中编写内容
2. 点击"文件" → "导出" → "公众号格式"
3. 看到"已复制公众号格式到剪贴板"提示
4. 打开微信公众号编辑器
5. 在编辑器中按 `Ctrl+V`（或 `Cmd+V`）粘贴
6. 格式会自动保留（标题、列表、粗体等）

   - 某些 CSS 样式可能不被支持

3. **格式调整**：
   - 粘贴后可能需要微调格式
   - 建议在公众号编辑器中预览

## 错误处理

### 如果复制仍然失败

1. **检查浏览器权限**
   - 确保浏览器允许访问剪贴板
   - 在浏览器设置中检查权限

2. **手动复制**
   - 如果自动复制失败，可以手动选择预览区内容
   - 右键 → 复制
   - 或使用 `Ctrl+A` 全选，然后 `Ctrl+C` 复制

3. **使用导出 HTML**
   - 作为替代方案，可以导出为 HTML 文件
   - 然后在浏览器中打开并复制内容

## 技术细节

### 为什么需要隐藏 textarea

```javascript
textarea.style.position = 'fixed'  // 固定定位，不影响布局
textarea.style.opacity = '0'       // 完全透明，用户看不到
```

这样可以：
- 不干扰页面布局
- 不影响用户体验
- 在后台静默完成复制

### execCommand 的兼容性

虽然 `document.execCommand()` 已被标记为废弃，但：
- 所有主流浏览器仍然支持
- 是最可靠的降级方案
- 在可预见的未来仍会继续工作

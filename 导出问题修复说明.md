# 导出问题修复说明

## 修复时间
2025年3月10日

## 问题描述
构建后的 fpk 包中，导出图片、表格和图表有问题，但开发模式正常。

## 根本原因
在导出功能中，代码依赖 `previewRef.current` 来获取预览区的 HTML 内容。
在某些情况下（特别是构建后的生产环境），`previewRef.current` 可能为 null 或 undefined，导致导出失败。

## 修复内容

### 1. App.jsx 修复
**位置**: 第 2527 行 - `convertToWechatFormat` 函数

**修复前**:
```javascript
const convertToWechatFormat = async (html) => {
  const container = document.createElement('div')
  container.innerHTML = html
  // ... 直接使用 previewRef.current
  const previewMathElements = previewRef.current.querySelectorAll('.katex, .katex-display')
```

**修复后**:
```javascript
const convertToWechatFormat = async (html) => {
  // 安全检查：确保 previewRef 可用
  if (!previewRef.current) {
    console.error('[微信导出] previewRef.current 不存在，返回原始 HTML')
    return html
  }
  
  const container = document.createElement('div')
  container.innerHTML = html
  
  // 安全获取数学公式元素
  let previewMathElements = []
  try {
    if (previewRef.current) {
      previewMathElements = previewRef.current.querySelectorAll('.katex, .katex-display')
    }
  } catch (err) {
    console.error('[微信导出] 获取预览区数学公式失败:', err)
  }
```

**位置**: 第 5474 行 - ExportDialog 组件调用

**修复前**:
```javascript
previewHtml={previewRef.current?.innerHTML}
```

**修复后**:
```javascript
previewHtml={previewRef.current?.innerHTML || ''}
```

### 2. ExportDialog.jsx 修复

**位置**: 第 205 行 - `exportAsHTML` 函数

**添加检查**:
```javascript
// 检查 previewHtml 是否为空
if (!previewHtml || previewHtml.trim() === '') {
  console.error('=== 导出错误 ===');
  console.error('previewHtml 为空，无法导出');
  alert('导出失败：预览内容为空。请确保文档已正确渲染。');
  return;
}
```

**位置**: 第 297 行 - `exportAsPDF` 函数

**添加检查**:
```javascript
// 检查 previewHtml 是否为空
if (!previewHtml || previewHtml.trim() === '') {
  console.error('=== 导出错误 ===');
  console.error('previewHtml 为空，无法导出 PDF');
  setError('导出失败：预览内容为空。请确保文档已正确渲染。');
  return;
}
```

## 修复效果

✅ **防止崩溃**: 即使 previewRef 为 null，也不会导致应用崩溃
✅ **友好提示**: 当导出失败时，会显示明确的错误信息
✅ **调试信息**: 添加了详细的 console.log，便于排查问题
✅ **向后兼容**: 不影响开发模式的正常使用

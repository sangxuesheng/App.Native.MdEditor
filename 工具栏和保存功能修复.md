# 工具栏和保存功能修复

## 🐛 问题描述

### 问题1: 工具栏禁用
- **现象**: 打开编辑器时工具栏按钮是灰色的（禁用状态）
- **原因**: 工具栏的禁用条件是 `disabled={!editorRef.current}`
- **影响**: 需要先编辑一下才能使用工具栏，即使光标在编辑区也不行

### 问题2: 无法保存
- **现象**: 打开编辑器直接编辑时，保存按钮是禁用的
- **原因**: 保存按钮的禁用条件是 `disabled={!currentPath}`
- **影响**: 新建文档时无法保存内容

---

## ✅ 修复方案

### 修复1: 工具栏始终启用

**修改前**:
```jsx
<EditorToolbar 
  onInsert={handleToolbarInsert}
  onImageUpload={handleImageUpload}
  onOpenImageManager={() => setShowImageManager(true)}
  onOpenTableInsert={() => setShowTableInsert(true)}
  disabled={!editorRef.current}  // ❌ 编辑器初始化后才启用
/>
```

**修改后**:
```jsx
<EditorToolbar 
  onInsert={handleToolbarInsert}
  onImageUpload={handleImageUpload}
  onOpenImageManager={() => setShowImageManager(true)}
  onOpenTableInsert={() => setShowTableInsert(true)}
  disabled={false}  // ✅ 始终启用
/>
```

### 修复2: 智能保存逻辑

**新增处理函数**:
```javascript
const handleSaveClick = () => {
  if (currentPath) {
    // 如果有路径，直接保存
    autoSave.manualSave()
  } else {
    // 如果没有路径，打开另存为对话框
    setShowSaveAsDialog(true)
  }
}
```

**修改保存按钮**:
```jsx
// 修改前
<button 
  className="btn-primary" 
  onClick={() => autoSave.manualSave()} 
  disabled={!currentPath}  // ❌ 没有路径时禁用
>
  保存
</button>

// 修改后
<button 
  className="btn-primary" 
  onClick={handleSaveClick} 
  disabled={false}  // ✅ 始终启用
>
  保存
</button>
```

## 🔧 技术细节

### 工具栏启用逻辑
- **之前**: 依赖 `editorRef.current` 是否存在
- **问题**: 编辑器初始化有延迟，导致工具栏短暂禁用
- **现在**: 始终启用，由工具栏内部处理编辑器未初始化的情况

### 保存逻辑
- **之前**: 依赖 `currentPath` 是否存在
- **问题**: 新建文档时无路径，保存按钮禁用
- **现在**: 智能判断
  - 有路径 → 直接保存
  - 无路径 → 打开另存为对话框

# 快速开发部署说明

## 开发模式 vs 生产模式

### 🚀 开发模式（推荐用于快速迭代）
使用 `appcenter-cli install-local` 直接从开发目录安装，**无需打包 fpk**。

**优点**：
- ⚡ 速度快，省去打包步骤
- 🔄 适合频繁修改和测试
- 🐛 便于调试

**步骤**：
```bash
# 1. 构建前端
cd /vol4/1000/开发文件夹/mac/app/ui/frontend
export PATH=/var/apps/nodejs_v22/target/bin:$PATH
npm run build

# 2. 直接从目录安装
cd /vol4/1000/开发文件夹/mac
appcenter-cli install-local
```

**一键部署**：
```bash
cd /vol4/1000/开发文件夹/mac && ./快速修复并部署.sh
```

### 📦 生产模式（用于发布）
打包成 fpk 文件后安装，用于正式发布。

**优点**：
- 📦 独立的安装包
- 🔒 版本固化
- 📤 便于分发

**步骤**：
```bash
# 1. 构建前端
cd /vol4/1000/开发文件夹/mac/app/ui/frontend
export PATH=/var/apps/nodejs_v22/target/bin:$PATH
npm run build

# 2. 打包 fpk
cd /vol4/1000/开发文件夹/mac
fnpack build

# 3. 安装 fpk
appcenter-cli install App.Native.MdEditor2.fpk
```

## 当前修复的部署流程

### 方式一：使用一键脚本（推荐）
```bash
cd /vol4/1000/开发文件夹/mac && ./快速修复并部署.sh
```

### 方式二：手动执行
```bash
# 1. 构建前端
cd /vol4/1000/开发文件夹/mac/app/ui/frontend
export PATH=/var/apps/nodejs_v22/target/bin:$PATH
npm run build

# 2. 停止旧版本
appcenter-cli stop App.Native.MdEditor2
sleep 2

# 3. 从本地目录安装
cd /vol4/1000/开发文件夹/mac
appcenter-cli install-local

# 4. 启动应用
appcenter-cli start App.Native.MdEditor2
sleep 5

# 5. 检查状态
curl http://localhost:18080/health
```

## 测试导出功能

1. 打开应用: http://192.168.2.2:18080/
2. 创建测试文档，包含：
   - 表格
   - 图片
   - Mermaid 图表
   - 数学公式
3. 点击"导出"按钮
4. 尝试导出为 HTML、PDF、PNG
5. 按 F12 打开浏览器控制台，查看调试信息

## 预期结果

✅ **成功情况**：
- 导出功能正常工作
- 表格、图片、图表都正确显示
- 控制台显示 `[微信导出]` 相关日志
- 没有红色错误信息

❌ **失败情况**：
- 显示"导出失败：预览内容为空"
- 控制台有红色错误信息
- 导出的文件内容为空

## 故障排查

如果导出仍然失败，请检查：

1. **浏览器控制台（F12）**：
   - 查找红色错误信息
   - 查找 `[微信导出]` 开头的日志
   - 查找 `=== 导出 HTML 调试信息 ===` 部分

2. **预览区是否正常渲染**：
   - 在编辑器中输入内容后，预览区是否显示
   - 切换到"仅预览"模式，检查内容是否完整

3. **网络请求**：
   - 检查 Network 标签
   - 确认 CSS、JS 资源是否加载成功
   - 特别注意 katex.min.css 和 highlight.js


## 开发技巧

### 热更新（仅修改前端代码）
如果只修改了前端代码（React 组件、CSS 等），可以使用热更新：

```bash
cd /vol4/1000/开发文件夹/mac/app/ui/frontend
export PATH=/var/apps/nodejs_v22/target/bin:$PATH
npm run build

# 重启应用即可
appcenter-cli restart App.Native.MdEditor2
```

### 完整重装（修改了后端或配置）
如果修改了后端代码（server.js）或配置文件（manifest），需要完整重装：

```bash
cd /vol4/1000/开发文件夹/mac && ./快速修复并部署.sh
```

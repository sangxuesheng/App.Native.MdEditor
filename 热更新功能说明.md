# 应用热更新功能说明

## ⚠️ 重要警告

**这些热更新方法不符合飞牛 NAS 官方规范，仅用于开发调试！**

- ❌ 不要在生产环境使用
- ❌ 可能导致应用不稳定
- ❌ 官方更新会覆盖热更新的文件
- ✅ 仅用于快速开发和测试

## 热更新方案对比

| 方案 | 更新内容 | 需要重启 | 速度 | 适用场景 |
|------|---------|---------|------|---------|
| 方案1：前端热更新 | 仅前端 | ❌ 否 | ⚡ 最快 | 前端样式/功能调试 |
| 方案2：后端热更新 | 仅后端 | ✅ 是 | 🚀 快 | 后端 API 调试 |
| 方案3：完整热更新 | 前端+后端 | ✅ 是 | 🐢 较慢 | 全栈功能开发 |
| 方案4：监听模式 | 自动前端 | ❌ 否 | ⚡ 实时 | 持续开发调试 |

## 方案 1：前端热更新（推荐）

### 特点
- ✅ 最快速，无需重启应用
- ✅ 只更新前端文件
- ✅ 适合 UI/样式调试

### 使用方法

```bash
# 赋予执行权限（首次使用）
chmod +x hot-update-frontend.sh

# 执行热更新
./hot-update-frontend.sh
```

### 工作流程
1. 构建前端（`npm run build`）
2. 备份当前前端文件
3. 替换到应用安装目录
4. 提示刷新浏览器

### 注意事项
- 更新后需要在浏览器中按 `Ctrl+Shift+R` 强制刷新
- 或清除浏览器缓存后刷新

## 方案 2：后端热更新

### 特点
- ✅ 更新后端代码
- ⚠️ 需要重启应用（约 5-10 秒）
- ✅ 适合 API 调试

### 使用方法

```bash
# 赋予执行权限（首次使用）
chmod +x hot-update-backend.sh

# 执行热更新
./hot-update-backend.sh
```

### 工作流程
1. 备份当前后端文件
2. 替换后端代码
3. 停止应用
4. 启动应用
5. 验证服务

## 方案 3：完整热更新

### 特点
- ✅ 同时更新前端和后端
- ⚠️ 需要重启应用
- ✅ 适合全栈开发
- ✅ 自动回滚功能

### 使用方法

```bash
# 赋予执行权限（首次使用）
chmod +x hot-update-full.sh

# 执行热更新
./hot-update-full.sh
```

### 工作流程
1. 构建前端
2. 备份前端和后端
3. 停止应用
4. 替换所有文件
5. 启动应用
6. 验证服务（失败自动回滚）

### 自动回滚
如果更新后服务启动失败，会自动回滚到之前的版本。

## 方案 4：监听模式（最高级）

### 特点
- ✅ 自动监听文件变化
- ✅ 自动构建和更新
- ✅ 实时开发体验
- ⚠️ 需要安装 inotify-tools

### 安装依赖

```bash
# 安装 inotify-tools
sudo apt-get update
sudo apt-get install inotify-tools
```

### 使用方法

```bash
# 赋予执行权限（首次使用）
chmod +x hot-update-watch.sh

# 启动监听模式
./hot-update-watch.sh
```

### 工作流程
1. 监听 `app/ui/frontend/src` 目录
2. 检测到文件变化
3. 自动构建前端
4. 自动替换文件
5. 提示刷新浏览器

### 停止监听
按 `Ctrl+C` 停止监听

### 防抖动
3 秒内只触发一次更新，避免频繁构建。

## 应用安装路径

飞牛 NAS 应用默认安装路径：
```
/var/packages/App.Native.MdEditor2/target/
```

目录结构：
```
target/
├── app/
│   └── server/          # 后端代码
└── ui/
    └── frontend/
        └── dist/        # 前端构建产物
```

## 备份管理

所有热更新都会自动创建备份：

```
backups/
├── hot-update-20260307_160000/          # 前端热更新备份
├── hot-update-backend-20260307_160100/  # 后端热更新备份
└── hot-update-full-20260307_160200/     # 完整热更新备份
```

### 手动回滚

如果需要回滚到之前的版本：

```bash
# 1. 停止应用
appcenter-cli stop App.Native.MdEditor2

# 2. 恢复备份文件
cp -r backups/hot-update-xxx/dist /var/packages/App.Native.MdEditor2/target/ui/frontend/

# 3. 启动应用
appcenter-cli start App.Native.MdEditor2
```

## 常见问题

### Q1: 权限不足

**问题**：执行脚本时提示权限不足

**解决**：
```bash
chmod +x hot-update-*.sh
```

### Q2: 应用路径不存在

**问题**：提示应用未安装或路径不存在

**解决**：
1. 确认应用已安装：`appcenter-cli list | grep MdEditor`
2. 检查路径：`ls /var/packages/App.Native.MdEditor2/`

### Q3: 更新后浏览器没变化

**问题**：热更新后浏览器显示还是旧版本

**解决**：
1. 强制刷新：`Ctrl+Shift+R`
2. 清除缓存：浏览器设置 → 清除缓存
3. 无痕模式：打开无痕窗口测试

### Q4: 服务启动失败

**问题**：热更新后服务无法启动

**解决**：
1. 查看日志：`tail -f /var/log/apps/App.Native.MdEditor2.log`
2. 手动回滚：使用备份文件恢复
3. 重新安装：`appcenter-cli install App.Native.MdEditor2.fpk`

### Q5: 监听模式不工作

**问题**：文件变化后没有自动更新

**解决**：
1. 检查 inotify-tools：`which inotifywait`
2. 检查监听目录：确保在项目根目录执行
3. 查看输出：检查是否有错误信息

## 开发工作流

### 推荐工作流（前端开发）

```bash
# 1. 启动监听模式（终端1）
./hot-update-watch.sh

# 2. 编辑代码（终端2）
vim app/ui/frontend/src/App.jsx

# 3. 保存文件
# 监听模式会自动构建和更新

# 4. 刷新浏览器
# Ctrl+Shift+R
```

### 推荐工作流（后端开发）

```bash
# 1. 编辑后端代码
vim app/server/server.js

# 2. 执行热更新
./hot-update-backend.sh

# 3. 测试 API
curl http://localhost:18080/api/xxx
```

### 推荐工作流（全栈开发）

```bash
# 1. 编辑前端和后端代码
vim app/ui/frontend/src/App.jsx
vim app/server/server.js

# 2. 执行完整热更新
./hot-update-full.sh

# 3. 测试功能
# 浏览器访问 http://localhost:18080
```

## 性能对比

|------|---------|-----------|---------|
| 前端更新 | 构建+打包+安装 (2-3分钟) | 构建+替换 (10-20秒) | **90%** |
| 后端更新 | 打包+安装 (1-2分钟) | 替换+重启 (10秒) | **85%** |
| 完整更新 | 构建+打包+安装 (3-5分钟) | 构建+替换+重启 (30秒) | **90%** |

## 安全建议

1. **仅在开发环境使用**
   - 不要在生产环境使用热更新
   - 生产环境应使用正式的 fpk 安装

2. **定期备份**
   - 热更新会自动创建备份
   - 定期清理旧备份节省空间

3. **测试后再发布**
   - 热更新测试通过后
   - 重新打包 fpk 正式发布

4. **版本控制**
   - 热更新的代码要提交到 Git
   - 确保代码可追溯

# 预览区图片缩放功能修复说明

## 🐛 问题描述

**原始问题**：在预览区右键点击图片并选择缩放时，编辑区的 Markdown 语法没有被替换成 HTML 语法。

### 问题原因分析

1. **预览区图片检测不完整**
   - `detectPreviewImage()` 函数只检测到预览区的图片元素
   - 但没有找到编辑区对应的 Markdown 语法位置
   - 导致无法定位和替换编辑区的内容

2. **缺少编辑器查找功能**
   - 预览区的图片和编辑区的语法之间没有建立关联
   - 需要通过图片的 `src` 属性在编辑器中查找对应的语法

---

## ✅ 解决方案

### 1. 新增 `findImageInEditor()` 函数

这个函数通过图片的 `src` 在编辑器中查找对应的 Markdown 或 HTML 语法：

```javascript
const findImageInEditor = useCallback((imageSrc) => {
  // 遍历编辑器所有行
  // 匹配 Markdown 语法: ![alt](src "title")
  // 匹配 HTML 语法: <img src="..." alt="..." />
  // 返回找到的图片信息和位置范围
}, [])
```

**功能特点**：
- ✅ 支持完整 URL 匹配
- ✅ 支持相对路径匹配
- ✅ 支持 Markdown 和 HTML 两种语法
- ✅ 返回精确的行号和列号范围

### 2. 修改 `handlePreviewContextMenu()` 函数

在预览区右键时，自动查找编辑区对应的语法：

```javascript
const handlePreviewContextMenu = useCallback((e) => {
  // 1. 检测预览区的图片
  const previewImage = detectPreviewImage(e.target)
  
  // 2. 在编辑器中查找对应的语法
  let selectedImage = null
  if (previewImage) {
    selectedImage = findImageInEditor(previewImage.src)
    // 合并预览区和编辑区的信息
    if (selectedImage) {
      selectedImage.alt = previewImage.alt || selectedImage.alt
      selectedImage.title = previewImage.title || selectedImage.title
    }
  }
  
  // 3. 设置右键菜单状态
  setContextMenu({ selectedImage, ... })
}, [detectPreviewImage, findImageInEditor])
```

## ⚠️ 注意事项

1. **URL 匹配**
   - 支持完整 URL 和相对路径
   - 使用 `includes()` 进行模糊匹配
   - 避免误匹配相似的 URL

2. **性能考虑**
   - 遍历所有行可能在大文件中较慢
   - 建议文档不超过 10000 行
   - 可以考虑添加缓存优化

3. **边界情况**
   - 如果编辑区有多个相同 src 的图片，会匹配第一个
   - 如果图片 URL 被修改，可能无法匹配
   - 建议使用唯一的图片 URL

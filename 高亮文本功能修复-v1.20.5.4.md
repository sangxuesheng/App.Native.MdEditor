# 高亮文本功能修复 v1.20.5.4

## 修复时间
2025-03-03

## 问题描述
使用 `==高亮文本==` 语法时，文本没有被正确渲染为高亮效果。

## 原因分析
`remarkGfm` 插件本身不支持 `==高亮==` 语法。这是一个扩展语法，需要额外处理。

## 解决方案

### 方法：预处理转换
在 Markdown 渲染之前，使用正则表达式将 `==文本==` 转换为 `<mark>文本</mark>` HTML 标签。

### 实现代码
```javascript
// 在 renderMarkdown 函数中添加预处理
const renderMarkdown = useCallback(async () => {
  if (!previewRef.current) return

  try {
    // 预处理：将 ==高亮== 转换为 <mark>高亮</mark>
    let processedContent = content.replace(/==([^=\n]+)==/g, '<mark>$1</mark>')
    
    // 使用 unified 处理器渲染 Markdown
    const processor = createMarkdownProcessor()
    const file = await processor.process(processedContent)
    let html = String(file)
    // ... 后续处理
  }
}, [content])
```

### 正则表达式说明
- `==` - 匹配开始标记
- `([^=\n]+)` - 捕获组，匹配非等号和非换行符的内容
- `==` - 匹配结束标记
- `g` - 全局匹配

### CSS 样式（已存在）

## 支持的用法

### 基础用法
```markdown
这是普通文本，==这是高亮文本==，继续普通文本。
```

### 嵌套样式
```markdown
==可以包含**粗体**文本==
==可以包含*斜体*文本==
==可以包含`代码`文本==
```

### 多个高亮
```markdown
==第一个高亮== 普通文本 ==第二个高亮== 普通文本 ==第三个高亮==
```

### 在列表中
```markdown
- ==列表项中的高亮==
- 普通列表项
```

### 在表格中
```markdown
| 表头1 | 表头2 |
|-------|-------|
| ==高亮单元格== | 普通单元格 |
```

### 在引用中
```markdown
> ==引用块中的高亮文本==
```

## 注意事项

### ✅ 正确写法
```markdown
==这是正确的高亮==
```

### ❌ 错误写法（有空格）
```markdown
== 这是错误的高亮 ==
```
**原因**: 正则表达式不匹配等号后的空格

### ⚠️ 限制
- 不支持跨行高亮（正则中使用了 `[^=\n]+`）
- 等号内部不能包含等号
